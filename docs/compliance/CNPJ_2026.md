## Sobre a Mudança

A **Instrução Normativa RFB nº 2229/2024** define que, a partir de **julho de 2026**, o CNPJ passará a aceitar letras nas posições 1 a 12, mantendo apenas os dois dígitos verificadores finais como numéricos.

### Motivação

| Razão | Detalhe |
|---|---|
| Esgotamento de CNPJs numéricos | Capacidade atual (~10¹⁴) insuficiente a longo prazo |
| Expansão de capacidade | Formato alfanumérico: 36¹² × 10² ≈ 4,7 × 10²¹ CNPJs |
| Compatibilidade retroativa | CNPJs numéricos existentes continuam válidos indefinidamente |

---

## Formatos

### Formato atual (até 2026)
```
Padrão: XX.XXX.XXX/XXXX-XX  (apenas dígitos 0-9)
Exemplo: 12.345.678/0001-95
```

### Formato novo (a partir de julho/2026)
```
Padrão: XX.XXX.XXX/XXXX-XX  (posições 1-12: dígitos OU letras A-Z; posições 13-14: dígitos verificadores)
Exemplo: AB.000.000/000X-58
```

**Estrutura das posições:**
- Posições 1-12: alfanumérico (0-9, A-Z maiúsculas)
- Posições 13-14: dígitos verificadores numéricos (calculados pelo algoritmo mod-11)

---

## Algoritmo dos Dígitos Verificadores (implementado)

O algoritmo é idêntico ao CNPJ numérico clássico (mod-11), com a adição de valores numéricos para letras:

| Caractere | Valor numérico |
|---|---|
| 0-9 | 0-9 (valor face) |
| A | 17 |
| B | 18 |
| C | 19 |
| … | … |
| Z | 42 |

**Cálculo:**
```
pesos1 = [5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2]   ← para dígito 1 (posições 1-12)
pesos2 = [6, 5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2] ← para dígito 2 (posições 1-13)

soma   = Σ valor(char[i]) × peso[i]
resto  = soma % 11
dígito = 0 se resto < 2, senão 11 - resto
```

## O Que Foi Implementado

### ✅ Backend — `core/validators.py`

```python
# Tabela de valores por caractere (0-9 = valor face; A=17 … Z=42)
_CNPJ_CHAR_VALUES: dict[str, int] = {str(i): i for i in range(10)}
_CNPJ_CHAR_VALUES.update({chr(ord('A') + i): 17 + i for i in range(26)})

def validar_cnpj(cnpj: str) -> None:
    """Valida CNPJ numérico clássico e alfanumérico 2026."""
    cnpj = re.sub(r'[^0-9A-Za-z]', '', cnpj).upper()  # normaliza

    if len(cnpj) != 14:
        raise ValidationError('CNPJ deve conter 14 caracteres.')

    if cnpj == cnpj[0] * 14:
        raise ValidationError('CNPJ inválido.')

    pesos1 = [5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2]
    pesos2 = [6, 5, 4, 3, 2, 9, 8, 7, 6, 5, 4, 3, 2]

    soma = sum(_CNPJ_CHAR_VALUES.get(cnpj[i], 0) * pesos1[i] for i in range(12))
    resto = soma % 11
    digito1 = 0 if resto < 2 else 11 - resto

    if int(cnpj[12]) != digito1:
        raise ValidationError('CNPJ inválido.')

    soma = sum(_CNPJ_CHAR_VALUES.get(cnpj[i], 0) * pesos2[i] for i in range(13))
    resto = soma % 11
    digito2 = 0 if resto < 2 else 11 - resto

    if int(cnpj[13]) != digito2:
        raise ValidationError('CNPJ inválido.')

def formatar_cnpj(cnpj: str) -> str:
    """Formata CNPJ alfanumérico: CC.CCC.CCC/CCCC-NN"""
    cnpj = re.sub(r'[^0-9A-Za-z]', '', cnpj).upper()
    if len(cnpj) != 14:
        return cnpj
    return f'{cnpj[:2]}.{cnpj[2:5]}.{cnpj[5:8]}/{cnpj[8:12]}-{cnpj[12:]}'

def validar_cpf_cnpj(valor: str, tipo: str = None) -> None:
    """
    Detecção automática: verifica 14 chars alfanuméricos (CNPJ) antes
    de 11 dígitos (CPF), evitando conflito quando CNPJ tem letras.
    """
    digits_only = re.sub(r'[^0-9]', '', valor)
    alnum_only  = re.sub(r'[^0-9A-Za-z]', '', valor)

    if tipo == 'PJ' or (tipo is None and len(alnum_only) == 14):
        validar_cnpj(valor)
    elif tipo == 'PF' or (tipo is None and len(digits_only) == 11):
        validar_cpf(valor)
    else:
        raise ValidationError('Documento deve ser CPF (11 dígitos) ou CNPJ (14 caracteres).')
```

### ✅ Frontend — `static/js/masks.js`

```javascript
function maskCnpj(v) {
    // Alphanumeric CNPJ 2026 (IN RFB 2229/2024): positions 1-12 alphanumeric, 13-14 numeric check digits
    var d = v.replace(/[^0-9A-Za-z]/g, '').toUpperCase().slice(0, 14);
    if (d.length <= 2) return d;
    if (d.length <= 5) return d.slice(0,2) + '.' + d.slice(2);
    if (d.length <= 8) return d.slice(0,2) + '.' + d.slice(2,5) + '.' + d.slice(5);
    if (d.length <= 12) return d.slice(0,2) + '.' + d.slice(2,5) + '.' + d.slice(5,8) + '/' + d.slice(8);
    return d.slice(0,2) + '.' + d.slice(2,5) + '.' + d.slice(5,8) + '/' + d.slice(8,12) + '-' + d.slice(12);
}
```

Comportamento da máscara:
- Aceita `[A-Za-z0-9]` — rejeita demais caracteres
- Auto-uppercase (minúsculas convertidas automaticamente)
- Aplica formatação `CC.CCC.CCC/CCCC-NN` progressivamente
