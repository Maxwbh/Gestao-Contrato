# HU-17 — Renegociação de Parcelas

## História do Usuário

**Como** operador da imobiliária  
**Quero** renegociar os termos de parcelas em aberto — ajustando valores, datas de vencimento ou condições — sem precisar cancelar e recriar o contrato  
**Para** atender acordos comerciais pós-contrato com o comprador de forma controlada e auditável

---

## Pré-condições

- Contrato com `status = ATIVO`
- Parcelas com `pago = False` elegíveis para renegociação
- Usuário com permissão de renegociação

---

## Critérios de Aceitação

1. `GET /financeiro/contrato/<contrato_id>/renegociar/` → interface com parcelas renegociáveis e campos de ajuste
2. `POST /financeiro/contrato/<contrato_id>/renegociar/` → aplica renegociação com as alterações informadas
3. Campos disponíveis para renegociação por parcela:
   - Novo `valor_atual` (pode ser menor que o original com desconto acordado)
   - Nova `data_vencimento`
   - Novo `percentual_juros_mora` ou `percentual_multa` temporário
4. Registro de auditoria criado com: usuário, data, motivo, valores antes/depois
5. `GET /contratos/<contrato_id>/renegociacao/historico/` → histórico de renegociações do contrato (se implementado)
6. Renegociação não altera `valor_original` — apenas `valor_atual`, preservando histórico

---

## Regras de Negócio

- RN-01: Parcelas já pagas não podem ser renegociadas
- RN-02: `valor_original` nunca é alterado — somente `valor_atual` para preservar histórico
- RN-03: A renegociação deve registrar os valores anteriores para auditoria completa
- RN-04: Boletos já emitidos para as parcelas renegociadas devem ser cancelados e regenerados com novos valores
- RN-05: Renegociação não interfere na lógica de bloqueio de reajuste — é independente do ciclo

---

## Definição de Pronto

- [ ] Interface exibe parcelas elegíveis com campos editáveis
- [ ] Valores antes/depois registrados para auditoria
- [ ] `valor_original` preservado; apenas `valor_atual` alterado
- [ ] Boletos existentes cancelados e regenerados
- [ ] Usuário executor registrado na auditoria

---

## Cenários de Teste

### CT-01 — Renegociar data de vencimento
**Dado** parcela vencendo em 10/jan, comprador solicitou prorrogação para 10/fev  
**Quando** operador altera `data_vencimento = 10/fev` e confirma  
**Então** parcela atualizada, registro de auditoria criado, boleto anterior cancelado

### CT-02 — Renegociar valor com desconto
**Dado** parcela com `valor_atual = R$ 1.500`, desconto acordado de R$ 200  
**Quando** define `valor_atual = R$ 1.300`  
**Então** `valor_atual = 1.300`, `valor_original` permanece R$ 1.500, auditoria registra diferença

### CT-03 — Parcela paga — não renegociável
**Dado** parcela com `pago = True`  
**Quando** tenta renegociar  
**Então** parcela não aparece na interface ou retorna erro

---

## Fluxo do Processo

```
Operador acessa "Renegociar Parcelas"
    │
    ▼
GET /financeiro/contrato/<id>/renegociar/
    │   Lista parcelas NORMAL não pagas
    │   Campos editáveis: valor_atual, data_vencimento
    │
    ▼
POST /financeiro/contrato/<id>/renegociar/
    │
    ├── Para cada parcela selecionada:
    │       Registrar: {valor_antes, data_antes, valor_depois, data_depois}
    │       parcela.valor_atual = novo_valor
    │       parcela.data_vencimento = nova_data
    │       parcela.save()
    │
    ├── Criar registro de auditoria:
    │       Renegociacao.create(contrato, usuario, data, motivo, parcelas_alteradas)
    │
    └── Se parcela tinha boleto gerado:
            cancelar_boleto(parcela)
            gerar_boleto(parcela, conta_bancaria)
    │
    ▼
Renegociação aplicada ✅
```
