# HU-07 — Gerar Carnê (Lote de Boletos)

## História do Usuário

**Como** operador da imobiliária  
**Quero** gerar um carnê em PDF com múltiplos boletos de um contrato de uma só vez  
**Para** entregar ao comprador um conjunto de boletos para pagamento nos próximos meses, sem precisar gerar um por um

---

## Pré-condições

- Contrato `ATIVO` com parcelas não pagas
- Conta bancária configurada na imobiliária
- Sem bloqueio de reajuste ativo para as parcelas a incluir (ou uso de `force`)
- API `boleto_cnab_api` disponível

---

## Critérios de Aceitação

1. `GET /financeiro/contrato/<id>/gerar-carne/` exibe interface de seleção de parcelas com campo de quantidade (padrão 20, mínimo 1)
2. `POST /financeiro/contrato/<id>/gerar-carne/` gera os boletos e retorna JSON `{sucesso, gerados, erros}`
3. Seleção das parcelas: próximas N com `pago = False`, ordenadas por `data_vencimento`
4. Para cada parcela selecionada: verifica bloqueio via `pode_gerar_boleto()` — se bloqueado, aborta com mensagem clara
5. Chamada em lote via `BoletoService.gerar_boletos_lote()` em chunks de 15 boletos por request à API
6. Falha em uma parcela individual não aborta o lote inteiro — erro é registrado e as demais são processadas
7. `GET /financeiro/contrato/<id>/carne/pdf/` → download do PDF concatenado com todos os boletos
8. `GET /financeiro/contrato/<id>/boletos/zip/` → ZIP contendo PDF individual de cada boleto
9. `POST /financeiro/api/carne/multiplos/` → ZIP com carnês de múltiplos contratos
10. Quando restam < N parcelas, gera apenas as disponíveis (sem erro)
11. Aviso informativo quando o reajuste está próximo (ex.: ciclo vence em < 3 meses) — informativo, não bloqueante

---

## Regras de Negócio

- RN-01: Parcelas pagas e canceladas são ignoradas na seleção
- RN-02: Quantidade padrão de 20 meses é o ciclo recomendado para reajuste anual
- RN-03: Quantidade de 6 meses é usada para imobiliárias que reemitem carnê semestralmente
- RN-04: Boletos gerados em lote são processados em chunks de 15 para evitar timeout
- RN-05: Cada boleto salva `boleto_pdf_db` (BinaryField) além do `boleto_pdf` (FileField)
- RN-06: `status_boleto = GERADO` e `nosso_numero` atribuído para cada boleto bem-sucedido

---

## Definição de Pronto

- [ ] PDF de carnê disponível para download com boletos numerados
- [ ] Cada parcela incluída com `status_boleto = GERADO` e `nosso_numero`
- [ ] Erros parciais reportados ao usuário (quais parcelas falharam e por quê)
- [ ] ZIP com PDFs individuais funcional
- [ ] Bloqueio de reajuste impede geração com mensagem clara

---

## Cenários de Teste

### CT-01 — Carnê de 20 meses sem bloqueio
**Dado** contrato com 60 parcelas restantes, sem reajuste pendente  
**Quando** `POST` com quantidade=20  
**Então** 20 boletos gerados, PDF disponível para download

### CT-02 — Carnê bloqueado por reajuste
**Dado** parcela na posição 13 (ciclo 2) com reajuste pendente  
**Quando** tenta gerar carnê com 20 parcelas (inclui parcela 13)  
**Então** retorna erro "Reajuste do ciclo 2 pendente. Aplique antes de gerar boletos."

### CT-03 — Menos parcelas que o solicitado
**Dado** contrato com apenas 5 parcelas restantes e quantidade=20  
**Quando** gera o carnê  
**Então** gera 5 boletos sem erro

### CT-04 — Falha parcial (1 parcela falha, demais ok)
**Dado** parcela 8 com dados de banco inválidos  
**Quando** gera carnê de 20 parcelas  
**Então** 19 boletos gerados com sucesso; resposta inclui lista de erros para a parcela 8

### CT-05 — Download ZIP de boletos individuais
**Dado** carnê gerado com 10 boletos  
**Quando** `GET /financeiro/contrato/<id>/boletos/zip/`  
**Então** ZIP com 10 PDFs individuais, cada um nomeado com o número da parcela

---

## Fluxo do Processo

```
Operador acessa "Gerar Carnê"
    │
    ▼
GET /financeiro/contrato/<id>/gerar-carne/
    │   Interface: selecionar quantidade (6 ou 20 padrão), filtrar parcelas
    │
    ▼
POST /financeiro/contrato/<id>/gerar-carne/
    │
    ├── Buscar próximas N parcelas não pagas (ordenadas por data_vencimento)
    │
    ├── Para cada parcela:
    │       Verificar pode_gerar_boleto() → se False: abortar com motivo
    │
    ├── Separar em chunks de 15:
    │       Chunk 1: parcelas 1–15 → POST /api/boleto/multi
    │       Chunk 2: parcelas 16–20 → POST /api/boleto/multi
    │
    ├── Para cada boleto gerado:
    │       parcela.nosso_numero = ...
    │       parcela.boleto_pdf_db = <PDF>
    │       parcela.status_boleto = GERADO
    │       parcela.save()
    │
    └── Concatenar PDFs → arquivo de carnê
    │
    ▼
GET /financeiro/contrato/<id>/carne/pdf/
    │   Serve PDF concatenado
    │
    ▼
GET /financeiro/contrato/<id>/boletos/zip/
    │   ZIP com PDFs individuais
    │
    ▼
Carnê disponível para download ✅
```
