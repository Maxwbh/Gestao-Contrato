# HU-23 — Envio Mensal de Remessa CNAB (Fluxo da Contadora)

## História do Usuário

**Como** contadora responsável pela cobrança de uma imobiliária  
**Quero** gerar e enviar os arquivos de remessa bancária dos boletos do mês em uma única tela simplificada  
**Para** registrar todos os boletos vencíveis no período junto ao banco, sem precisar entender os layouts CNAB ou selecionar boleto por boleto

---

## Nova Persona

| Persona | Descrição |
|---------|-----------|
| **Contadora** | Usuária operacional que não conhece CNAB mas precisa, todo início de mês, gerar boletos dos contratos vigentes e enviar os arquivos de remessa a um ou mais bancos. Valoriza clareza, velocidade e confirmação visual do que foi processado. |

> **Relação com personas existentes**: A Contadora pode executar as mesmas ações do Operador nas telas de Carnê e Remessa, mas entra pela rota simplificada documentada nesta HU.

---

## Pré-condições

- Pelo menos um contrato ativo com parcelas sem boleto gerado **ou** boletos com `status_boleto = GERADO` cujo `nosso_numero` ainda não entrou em nenhuma remessa com `status = ENVIADO`
- `data_vencimento >= date.today()` (só parcelas presentes e futuras)
- Conta bancária configurada na imobiliária com convênio, agência, conta e `carteira`
- API `boleto_cnab_api` disponível para geração de boletos

---

## Critérios de Aceitação

### Passo 1 — Visão do Mês (Painel de Remessa)

1. `GET /financeiro/remessa/` → página "Envio de Remessa do Mês" com:
   - **Painel de resumo** no topo: total de contratos ativos, total de parcelas elegíveis, total de boletos pendentes de remessa, valor total envolvido
   - **Seletor de período** (mês/ano, default = mês atual) que re-filtra os dados sem recarregar a página
   - **Grupos por banco/layout**: cada conta bancária com remessa pendente aparece como card separado, exibindo:
     - Nome do banco + número da conta
     - Formato CNAB (240 ou 400)
     - Quantidade de boletos elegíveis no grupo
     - Valor total do grupo
     - Botão "Gerar Remessa" por grupo

2. Boleto é **elegível** para remessa se:
   - `status_boleto = GERADO`
   - `nosso_numero ≠ ''`
   - `data_vencimento >= date.today()`
   - Não existe `ItemRemessa` apontando para este boleto em uma remessa com `status = ENVIADO` ou `PROCESSADO`

3. Se não houver boletos gerados mas houver parcelas sem boleto, exibir botão de atalho **"Gerar Boletos do Mês"** que direciona para `/financeiro/parcelas/?mes=<atual>&status_boleto=NAO_GERADO`

### Passo 2 — Geração dos Arquivos por Banco

4. `POST /financeiro/remessa/gerar/` (corpo: `{conta_bancaria_id, mes, ano}`) → gera o arquivo `.rem` para aquele grupo:
   - Filtra boletos elegíveis da `conta_bancaria_id` no período
   - Chama `CNABService.gerar_remessa(boletos, conta_bancaria)`
   - Cria `ArquivoRemessa` com `status = GERADO` e retorna JSON `{arquivo_id, total_boletos, valor_total, download_url}`
   - O botão "Gerar Remessa" vira "Baixar Arquivo .rem" ao concluir, sem recarregar a página

5. `GET /financeiro/remessa/<pk>/download/` → serve o arquivo `.rem` para download direto

### Passo 3 — Marcação como Enviado

6. `POST /financeiro/remessa/<pk>/marcar-enviada/` → muda `status = ENVIADO`, grava `data_envio`, retorna JSON `{status: 'ENVIADO', data_envio}`
   - O card do banco atualiza o badge de status em tempo real (sem reload)
   - Botão "Marcar como Enviada" fica disponível somente após o download ser realizado ao menos uma vez (`data_geracao IS NOT NULL`)

### Painel de Histórico

7. `GET /financeiro/remessa/historico/` → tabela paginada de todos os `ArquivoRemessa`:
   - Colunas: data, banco, layout, nº boletos, valor total, status (badge colorido), ações (baixar / reenviar)
   - Filtros: mês/ano, banco, status
   - Badge de status: `GERADO` (cinza), `ENVIADO` (azul), `PROCESSADO` (verde), `ERRO` (vermelho)

8. Quando todos os grupos de um mês estiverem com `status ∈ {ENVIADO, PROCESSADO}`, exibir banner de conclusão: "Remessa de [Mês/Ano] concluída — X arquivos enviados para Y bancos"

---

## Regras de Negócio

- **RN-01**: Parcela com `data_vencimento < date.today()` **não entra** na remessa — remessa é sempre presente/futuro; boletos vencidos devem ser tratados via renegociação ou segunda via
- **RN-02**: Cada `ContaBancaria` gera **um arquivo por mês** — se a contadora gerar novamente no mesmo mês, o sistema reutiliza o `ArquivoRemessa` existente (se `status = GERADO`) ou cria um adicional com numeração sequencial (`numero_remessa` auto-increment)
- **RN-03**: Boletos que já estão em uma remessa `ENVIADA` ou `PROCESSADA` não entram em uma nova remessa — evita registro duplicado no banco
- **RN-04**: Um boleto sem `nosso_numero` é silenciosamente excluído da remessa e aparece no sumário como "X boletos ignorados (sem nosso número)" com link para o relatório
- **RN-05**: Cada banco usa o layout configurado em `ContaBancaria.layout_cnab` (`CNAB_240` ou `CNAB_400`); o `CNABService` seleciona o formato automaticamente — a contadora não escolhe o formato
- **RN-06**: O status `ENVIADO` é marcação **manual** — o sistema não envia o arquivo ao banco automaticamente; a contadora baixa o `.rem` e envia pelo internet banking do banco
- **RN-07**: Reprocessar uma remessa com `status = GERADO` sobrescreve o arquivo anterior; `status = ENVIADO` bloqueia regeneração (exibe aviso: "Remessa já enviada — para regenerar, cancele o envio")
- **RN-08**: Herda **RN-G08** (idempotência de retorno): o retorno bancário que chega depois não duplica pagamentos já confirmados
- **RN-09**: `ArquivoRemessa.numero_remessa` é seqüencial por `ContaBancaria` — incrementado automaticamente a cada nova geração

## Cenários de Teste

### CT-01 — Painel lista grupos corretos por banco
**Dado** imobiliária com 2 contas bancárias (BB CNAB 240, Bradesco CNAB 400) e 5 boletos GERADO no BB e 3 no Bradesco com vencimento futuro  
**Quando** `GET /financeiro/remessa/`  
**Então** exibe 2 cards: "BB — 5 boletos — R$ X.XXX,XX" e "Bradesco — 3 boletos — R$ Y.YYY,YY"

### CT-02 — Boleto vencido não aparece
**Dado** 4 boletos GERADO, sendo 1 com `data_vencimento = ontem`  
**Quando** acessa o painel  
**Então** grupo exibe 3 boletos; o vencido não está listado

### CT-03 — Gerar remessa cria ArquivoRemessa e retorna link de download
**Dado** grupo com 3 boletos elegíveis  
**Quando** `POST /financeiro/remessa/gerar/` com `{conta_bancaria_id: X, mes: 6, ano: 2026}`  
**Então** JSON `{arquivo_id: N, total_boletos: 3, download_url: '/financeiro/remessa/N/download/'}`, `ArquivoRemessa` com status GERADO e 3 `ItemRemessa`

### CT-04 — Download serve arquivo .rem
**Dado** `ArquivoRemessa` criado  
**Quando** `GET /financeiro/remessa/<pk>/download/`  
**Então** resposta com `Content-Type: application/octet-stream` e `Content-Disposition: attachment; filename="remessa_<numero>.rem"`

### CT-05 — Marcar como enviada bloqueia regeneração
**Dado** remessa com `status = ENVIADO`  
**Quando** tenta `POST /financeiro/remessa/gerar/` novamente para o mesmo mês/conta  
**Então** JSON `{erro: 'Remessa já enviada para este banco neste mês.'}`, status HTTP 409; nenhum arquivo sobrescrito

### CT-06 — Boleto já em remessa ENVIADA não entra em nova remessa
**Dado** boleto B com `nosso_numero` em uma `ItemRemessa` da remessa R (status ENVIADO)  
**Quando** gera nova remessa para a mesma conta  
**Então** boleto B não aparece no novo arquivo; `total_boletos` exclui B

### CT-07 — Banner de conclusão ao finalizar todos os grupos
**Dado** imobiliária com 2 contas bancárias, ambas com remessa ENVIADA no mês atual  
**Quando** acessa `/financeiro/remessa/`  
**Então** banner verde: "Remessa de Junho/2026 concluída — 2 arquivos enviados para 2 bancos"

### CT-08 — Múltiplos bancos com layouts diferentes
**Dado** BB (CNAB 240) e Itaú (CNAB 400) com boletos elegíveis  
**Quando** gera remessa para cada banco  
**Então** arquivo do BB usa layout CNAB 240; arquivo do Itaú usa layout CNAB 400; ambos são independentes

### CT-09 — Parcela sem nosso_numero excluída com aviso
**Dado** 4 boletos elegíveis, sendo 1 sem `nosso_numero`  
**Quando** gera remessa  
**Então** arquivo com 3 registros; JSON inclui `{ignorados: 1, motivo: 'sem_nosso_numero'}`

## Fluxo do Processo

```
CONTADORA ENTRA NO PAINEL DE REMESSA
    │
    ▼
GET /financeiro/remessa/?mes=06&ano=2026
    │
    ├── Se não há boletos GERADO com vencimento futuro:
    │       Exibe: "Nenhum boleto pronto para remessa"
    │       Botão: "Gerar Boletos do Mês" → /financeiro/parcelas/?mes=atual&status_boleto=NAO_GERADO
    │
    └── Se há boletos elegíveis:
            Agrupa por (conta_bancaria, layout_cnab)
            Renderiza um card por grupo
    │
    ▼ (para cada banco)
[Card do Banco]
    ├── "Banco do Brasil — CNAB 240 — 12 boletos — R$ 18.400,00"
    │       [Botão: Gerar Remessa]
    │
    ▼ (clica em "Gerar Remessa")
POST /financeiro/remessa/gerar/
    │   body: {conta_bancaria_id, mes, ano}
    │
    ├── Filtrar: status_boleto = GERADO
    │           nosso_numero != ''
    │           data_vencimento >= hoje
    │           sem ItemRemessa em remessa ENVIADA/PROCESSADA
    │
    ├── CNABService.gerar_remessa(boletos, conta_bancaria)
    │       → Seleciona layout (CNAB_240 ou CNAB_400) via conta_bancaria.layout_cnab
    │       → Monta arquivo .rem
    │       → Calcula dígitos verificadores
    │
    ├── ArquivoRemessa.create(arquivo=bytes, status=GERADO, total_registros=N)
    ├── ItemRemessa.bulk_create([...])
    │
    └── Retorna JSON: {arquivo_id, total_boletos, valor_total, download_url}
    │
    ▼ (botão muda para "Baixar Arquivo .rem")
GET /financeiro/remessa/<pk>/download/
    │   Content-Disposition: attachment; filename="remessa_<numero>.rem"
    │
    ▼ (contadora faz upload no internet banking do banco)

[Após confirmar envio ao banco]
    │
    ▼
POST /financeiro/remessa/<pk>/marcar-enviada/
    │   status = ENVIADO, data_envio = now()
    │   Badge do card atualiza para "ENVIADO" (azul)
    │
    ▼ (repete para cada banco com boletos pendentes)

TODOS OS BANCOS MARCADOS COMO ENVIADOS
    │
    ▼
Banner: "Remessa de Junho/2026 concluída — X arquivos enviados para Y bancos" ✅
    │
    ▼
[Dias depois — retorno bancário chega como .ret]
    POST /financeiro/cnab/retorno/upload/ → processa via HU-16
```

## Referências e Dependências

| Referência | Relação |
|-----------|---------|
| **HU-03** (Gerar Boleto Individual) | Pré-requisito: boletos precisam estar GERADO antes da remessa |
| **HU-07** (Gerar Carnê) | Alternativa para gerar todos os boletos de um contrato de uma vez |
| **HU-16** (CNAB — Remessa e Retorno) | Esta HU estende HU-16 com a rota simplificada; HU-16 documenta o processamento de retorno |
| `ArquivoRemessa`, `ItemRemessa` | Modelos persistidos em `financeiro/models.py` |
| `CNABService` | Serviço em `financeiro/services/cnab_service.py` — seleção de layout por banco |
| `ContaBancaria.layout_cnab` | Campo `('CNAB_240', 'CNAB_400')` determina formato por conta |

### Estados do Card de Banco

| Estado | Badge | Botões Disponíveis |
|--------|-------|--------------------|
| Boletos disponíveis, sem remessa | `PENDENTE` (amarelo) | "Gerar Remessa" |
| Arquivo gerado, não baixado | `GERADO` (cinza) | "Baixar .rem", "Regenerar" |
| Arquivo baixado | `GERADO` (cinza) | "Baixar .rem", "Marcar como Enviada", "Regenerar" |
| Marcada como enviada | `ENVIADO` (azul) | "Baixar .rem" (somente leitura) |
| Retorno processado | `PROCESSADO` (verde) | — (somente leitura) |
