# HU-22 — Mapa Interativo de Lotes

## História do Usuário

**Como** gestor ou operador da imobiliária  
**Quero** visualizar todos os imóveis do portfólio em um mapa interativo com marcadores por status  
**Para** entender rapidamente a distribuição geográfica do estoque, identificar lotes disponíveis e acompanhar a evolução das vendas por empreendimento

---

## Pré-condições

- Imóveis cadastrados com coordenadas `latitude` e `longitude`
- Usuário autenticado com acesso à imobiliária

---

## Critérios de Aceitação

### Mapa Geral — Lista de Imóveis

1. `GET /imoveis/` exibe mapa Leaflet com todos os imóveis que possuem coordenadas (não paginado — todos passados ao mapa de uma vez)
2. Marcadores customizados via `DivIcon`: círculo **verde** = disponível, círculo **vermelho** = vendido
3. Agrupamento automático com `Leaflet.markercluster` — marcadores próximos colapsados em cluster com contagem
4. Popup ao passar o mouse (`hover`) exibe: nome do imóvel, loteamento, status e link para detalhes
5. Switcher de camadas no canto superior direito: Carto Voyager (padrão), Esri Satellite, Carto Dark
6. Filtro por loteamento (dropdown JavaScript, sem reload de página) — atualiza marcadores visíveis
7. Filtro por status (disponível / vendido) — combinável com filtro de loteamento
8. Legenda inline no canto inferior direito (disponível ● / vendido ●)
9. Contador dinâmico exibe quantos marcadores estão visíveis após aplicar filtros
10. Botão "Perto de mim" usa `navigator.geolocation` → centraliza mapa na posição do usuário + marcador azul "Você está aqui" + lista lotes num raio de 50 km

### Página por Loteamento

11. `GET /imoveis/loteamento/<nome>/` exibe mapa dedicado do empreendimento com lista filtrável por status
12. KPI cards do loteamento: total de lotes, disponíveis, vendidos, valor médio/min/max, barra de progresso proporcional
13. Links "Ver no Google Maps" e "Waze" no popup quando `lat/lng` disponíveis
14. Acesso restrito ao loteamento da imobiliária do usuário autenticado

---

## Regras de Negócio

- RN-01: Imóveis sem coordenadas não aparecem no mapa — ausência de `lat/lng` não gera erro
- RN-02: Todos os imóveis com coordenadas são passados ao mapa em uma única query (sem paginação) para permitir clustering client-side
- RN-03: Tiles Carto Voyager usados como padrão — sem API key necessária (gratuito)
- RN-04: Filtros de loteamento e status funcionam exclusivamente no client-side (JavaScript) — sem request ao servidor
- RN-05: Geolocalização é opcional — funciona apenas com permissão do browser; ausência de permissão não quebra o mapa

---

## Definição de Pronto

- [ ] Mapa renderizado com todos os imóveis geocodificados
- [ ] Marcadores verde/vermelho por status de disponibilidade
- [ ] Clustering ativo com contagem de marcadores
- [ ] Filtros de loteamento e status sem reload
- [ ] Switcher de camadas (Voyager / Satellite / Dark)
- [ ] Página dedicada por loteamento com KPIs e lista filtrável

## Cenários de Teste

### CT-01 — Mapa carregado com todos os imóveis
**Dado** 150 imóveis, 80 com coordenadas (50 disponíveis, 30 vendidos)  
**Quando** acessa `/imoveis/`  
**Então** mapa exibe 80 marcadores; 50 verdes, 30 vermelhos; imóveis sem coords ignorados

### CT-02 — Clustering de marcadores próximos
**Dado** 20 lotes no mesmo empreendimento com coordenadas próximas  
**Quando** mapa está no zoom padrão  
**Então** `markercluster` colapsa em 1 cluster com badge "20"; zoom in expande

### CT-03 — Filtro por loteamento sem reload
**Dado** loteamentos "Parque das Nogueiras" e "Jardim Europa" visíveis  
**Quando** seleciona "Parque das Nogueiras" no dropdown  
**Então** apenas marcadores do Parque das Nogueiras visíveis; contador atualiza; sem reload

### CT-04 — Filtro combinado loteamento + status
**Dado** filtro "Parque das Nogueiras" + "disponível"  
**Quando** filtros aplicados  
**Então** exibe apenas lotes disponíveis daquele loteamento; vermelhos ocultos

### CT-05 — Página de loteamento com KPIs
**Dado** loteamento com 100 lotes: 35 vendidos, 65 disponíveis; preços entre R$80k e R$150k  
**Quando** acessa `/imoveis/loteamento/parque-das-nogueiras/`  
**Então** KPIs: total=100, disponíveis=65, vendidos=35, valor médio≈R$115k, barra 35% vendido

### CT-06 — Geolocalização
**Dado** browser concede permissão de geolocalização  
**Quando** clica "Perto de mim"  
**Então** mapa centraliza na posição do usuário; marcador azul aparece; exibe contagem de lotes num raio de 50 km

## Fluxo do Processo

```
Operador acessa /imoveis/
    │
    ▼
Django serializa todos os imóveis com coords → JSON no contexto
    │
    ▼
Leaflet inicializado com tiles Carto Voyager
    │
    ├── Para cada imóvel com lat/lng:
    │       DivIcon verde (disponível) ou vermelho (vendido)
    │       Popup com nome, loteamento, status, link detalhes
    │       Adicionado ao MarkerClusterGroup
    │
    ├── Switcher de camadas:
    │       Voyager (padrão) / Satellite / Dark → trocam tiles sem reload
    │
    ├── Filtro por loteamento (dropdown JS):
    │       onChange → `.eachLayer()` → show/hide baseado em loteamento
    │       Atualiza contador de visíveis
    │
    ├── Filtro por status (disponível/vendido):
    │       onChange → `.eachLayer()` → show/hide combinado com filtro loteamento
    │
    └── Botão "Perto de mim":
            navigator.geolocation.getCurrentPosition()
            Centraliza mapa na posição
            Adiciona marcador azul "Você está aqui"
            Conta marcadores num raio de 50 km
    │
    ▼
/imoveis/loteamento/<nome>/
    │
    ├── Mapa dedicado do empreendimento
    ├── Lista filtrável por status (sem reload)
    └── KPI cards: total, disponíveis, vendidos, valor médio/min/max, progresso %
```
