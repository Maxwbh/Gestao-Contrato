# HU-15 — Importação de Índices Econômicos

## História do Usuário

**Como** operador da imobiliária  
**Quero** importar automaticamente os índices econômicos (IPCA, IGPM, INCC, SELIC, TR, INPC, IGPDI) das fontes oficiais  
**Para** que os cálculos de reajuste utilizem dados atualizados sem necessidade de digitação manual

---

## Pré-condições

- Conexão com internet para acesso às APIs IBGE, BCB e FGV
- Usuário com permissão de administrador de índices

---

## Critérios de Aceitação

1. `GET /contratos/indices/` → lista todos os `IndiceReajuste` cadastrados, filtrável por tipo e período
2. `GET /contratos/indices/novo/` → formulário para criação manual de um índice (mês, ano, valor)
3. `GET /contratos/indices/<pk>/editar/` → editar índice existente
4. `POST /contratos/indices/importar/` → dispara importação automática das APIs oficiais:
   - **IPCA** → API IBGE (série 433 — número-índice + variação mensal)
   - **IGPM** → BCB (série 189) com fallback para FGV
   - **INCC** → FGV / BCB (série 7459)
   - **SELIC** → BCB API (série 11)
   - **TR** → BCB (série 226)
   - **INPC** → IBGE (série 188)
   - **IGPDI** → FGV / BCB
5. Para cada índice importado, registra: `tipo_indice`, `ano`, `mes`, `valor` (variação %), `numero_indice` (número-índice IBGE/FGV quando disponível), `fonte`, `data_importacao`
6. Importação é **idempotente** — se o índice do mês já existe, atualiza apenas se o valor diverge
7. Resultado da importação exibe: quantidade importada, quantidade atualizada, erros por fonte
8. Cálculo acumulado via `IndiceReajuste.get_acumulado_periodo()`:
   - **Método 1** (prioritário): `% = numero_indice_fim / numero_indice_base − 1` (usa `numero_indice` IBGE/FGV)
   - **Método 2** (fallback): `∏(1 + valor_k/100) − 1` (produto das variações mensais)
9. `GET /financeiro/api/indice-reajuste/` → retorna índice por tipo/mês/ano (JSON)
10. `GET /financeiro/api/calcular-indice-acumulado/` → retorna acumulado do período informado (JSON)

---

## Regras de Negócio

- RN-01: `(tipo_indice, ano, mes)` é chave única — duplicata atualiza em vez de criar
- RN-02: `numero_indice` é usado no Método 1 de cálculo acumulado quando disponível — resulta em valor mais preciso que o produto das variações
- RN-03: Se a API oficial estiver indisponível, a importação registra o erro mas não aborta os demais índices
- RN-04: `tipo_correcao = FIXO` no contrato nunca consulta `IndiceReajuste`
- RN-05: A importação pode ser disparada manualmente ou por cron job periódico

---

## Definição de Pronto

- [ ] CRUD manual de índices funcional
- [ ] Importação automática conecta às APIs IBGE, BCB e FGV
- [ ] Idempotência: reimportar o mesmo mês não duplica registros
- [ ] `get_acumulado_periodo()` usa Método 1 quando `numero_indice` disponível
- [ ] Resultado da importação exibe totais e erros por fonte
- [ ] Endpoint JSON retorna índice e acumulado

## Cenários de Teste

### CT-01 — Importar IPCA do mês atual
**Dado** API IBGE disponível com dados de abr/2026  
**Quando** dispara importação  
**Então** `IndiceReajuste(tipo=IPCA, ano=2026, mes=4, valor=0.43, numero_indice=..., fonte=IBGE)` criado

### CT-02 — Reimportar mesmo mês — idempotência
**Dado** IPCA abril/2026 já importado com `valor=0.43`  
**Quando** importa novamente  
**Então** registro atualizado se valor mudou; nenhum duplicado criado

### CT-03 — API indisponível — não aborta demais índices
**Dado** API FGV fora do ar  
**Quando** importa todos os índices  
**Então** IPCA, SELIC, TR importados normalmente; IGPM e IGPDI registram erro; relatório mostra parcialmente

### CT-04 — Cálculo acumulado IPCA 12 meses via número-índice
**Dado** `numero_indice` de jan/2025 = 5.200 e jan/2026 = 5.680  
**Quando** `get_acumulado_periodo(IPCA, 2025, 1, 2026, 1)`  
**Então** `% = 5.680 / 5.200 − 1 = 9,23%` (Método 1)

### CT-05 — Cálculo acumulado — fallback Método 2
**Dado** `numero_indice` não disponível para o período  
**Quando** `get_acumulado_periodo(IGPM, 2024, 1, 2024, 12)`  
**Então** usa produto das variações mensais (Método 2)

## Fluxo do Processo

```
Operador clica "Importar Índices"
    │
    ▼
POST /contratos/indices/importar/
    │
    ├── Para cada tipo de índice [IPCA, IGPM, INCC, SELIC, TR, INPC, IGPDI]:
    │       │
    │       ├── IPCA → GET https://servicodados.ibge.gov.br/api/v3/agregados/433/...
    │       ├── IGPM → GET https://api.bcb.gov.br/dados/serie/bcdata.sgs.189/dados/...
    │       ├── SELIC → GET https://api.bcb.gov.br/dados/serie/bcdata.sgs.11/dados/...
    │       ├── TR   → GET https://api.bcb.gov.br/dados/serie/bcdata.sgs.226/dados/...
    │       └── ...
    │
    ├── Para cada valor recebido:
    │       IndiceReajuste.objects.update_or_create(
    │           tipo_indice=tipo, ano=ano, mes=mes,
    │           defaults={valor: variação, numero_indice: nro, fonte: fonte}
    │       )
    │
    └── Retorna: {importados: N, atualizados: N, erros: [{tipo, motivo}]}
    │
    ▼ (cálculo de reajuste — HU-05)
IndiceReajuste.get_acumulado_periodo(tipo, ano_ini, mes_ini, ano_fim, mes_fim)
    │
    ├── Se numero_indice disponível em inicio e fim:
    │       return numero_indice_fim / numero_indice_base − 1  (Método 1)
    │
    └── Else:
            return ∏(1 + valor_k/100) − 1  (Método 2)
```
