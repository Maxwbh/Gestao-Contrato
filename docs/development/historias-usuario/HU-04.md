# HU-04 — Pagamento de Parcela

## História do Usuário

**Como** operador da imobiliária  
**Quero** registrar o pagamento de uma parcela com os encargos corretos  
**Para** atualizar a posição financeira do contrato, dar baixa no boleto e, quando aplicável, quitar o contrato automaticamente

---

## Pré-condições

- Parcela existente com `pago = False`
- Usuário autenticado com permissão de registrar pagamentos

---

## Critérios de Aceitação

1. Formulário em `GET /financeiro/parcelas/<id>/pagar/` exibe: valor original, multa atual, juros atual, desconto disponível e total a pagar para hoje
2. `POST /financeiro/parcelas/<id>/pagar/` registra o pagamento com: `valor_pago`, `data_pagamento`, `forma_pagamento`
3. Formas de pagamento suportadas: PIX, BOLETO, CHEQUE, DINHEIRO, TRANSFERÊNCIA e outras
4. `POST /financeiro/parcelas/<id>/pagar-ajax/` permite registrar via AJAX sem reload de página
5. `GET /financeiro/parcelas/<id>/calcular-encargos/` retorna JSON com multa, juros e desconto para uma data informada
6. Cria `HistoricoPagamento` com `origem_pagamento = MANUAL`
7. Parcela marcada `pago = True` e `status_boleto = PAGO` após registro
8. Se todas as parcelas NORMAL + INTERMEDIARIA do contrato estiverem pagas, `contrato.status = QUITADO`
9. Reversão de pagamento disponível para usuários com permissão — cria novo `HistoricoPagamento` com `valor_pago` negativo

---

## Regras de Negócio

- RN-01: `data_pagamento` não pode ser futura
- RN-02: Cálculo de encargos por `Parcela.calcular_juros_multa(data_referencia)`:
  - Se `data_referencia ≤ data_vencimento` → juros = 0, multa = 0
  - Se atrasado: `multa = valor_atual × percentual_multa / 100` (uma vez)
  - Se atrasado: `juros = valor_atual × (percentual_juros_mora / 100) × (dias_atraso / 30)` (pro rata die)
- RN-03: Desconto disponível se `data_vencimento − dias_desconto ≤ hoje ≤ data_vencimento`
- RN-04: Valor com encargos = `valor_atual − desconto + juros + multa`
- RN-05: Pagamento via OFX exige `fitid_ofx` único para deduplicação (ver HU-10)
- RN-06: Registro CNAB associa `ItemRetorno` à parcela para rastreabilidade
- RN-07: Quitação do contrato é verificada somente após o pagamento; parcelas ENTRADA não bloqueiam quitação

---

## Definição de Pronto

- [ ] `HistoricoPagamento` persistido com `origem_pagamento = MANUAL`, data e valor corretos
- [ ] `parcela.pago = True` e `parcela.status_boleto = PAGO`
- [ ] Contrato marcado `QUITADO` quando todas as parcelas estão pagas
- [ ] Cálculo de encargos correto para qualquer data informada
- [ ] Reversão disponível sem apagar o histórico original

---

## Cenários de Teste

### CT-01 — Pagamento à vista sem encargos
**Dado** parcela com vencimento hoje  
**Quando** operador registra `valor_pago = valor_original`, `data_pagamento = hoje`, `forma_pagamento = PIX`  
**Então** `HistoricoPagamento` criado, `pago = True`, `status_boleto = PAGO`

### CT-02 — Pagamento em atraso com juros e multa
**Dado** parcela vencida há 15 dias, `percentual_juros_mora = 1%/mês`, `percentual_multa = 2%`  
**Quando** operador consulta encargos via API  
**Então** `multa = valor × 0,02`, `juros = valor × (0,01/30) × 15`, total = valor + multa + juros

### CT-03 — Pagamento final quita contrato
**Dado** todas as parcelas pagas exceto a última  
**Quando** operador registra pagamento da última parcela  
**Então** `contrato.status = QUITADO`

### CT-04 — Reversão de pagamento
**Dado** parcela marcada como paga por engano  
**Quando** usuário com permissão reverte o pagamento  
**Então** novo `HistoricoPagamento` com valor negativo, `parcela.pago = False`, `status_boleto = GERADO`

### CT-05 — Data futura rejeitada
**Dado** `data_pagamento = amanhã`  
**Quando** formulário é submetido  
**Então** validação rejeita com mensagem "Data de pagamento não pode ser futura"

## Fluxo do Processo

```
Operador acessa parcela
    │
    ▼
GET /financeiro/parcelas/<id>/pagar/
    │   Exibe: valor_atual, juros_hoje, multa_hoje, desconto_hoje, total_hoje
    │   (via Parcela.calcular_valores_hoje())
    │
    ▼
POST /financeiro/parcelas/<id>/pagar/
    │   Campos: valor_pago, data_pagamento, forma_pagamento, desconto, juros, multa
    │
    ├── Validar data_pagamento ≤ hoje
    ├── Calcular valor_com_encargos
    ├── Criar HistoricoPagamento(origem=MANUAL)
    ├── parcela.pago = True
    ├── parcela.status_boleto = PAGO
    ├── parcela.save()
    │
    └── Verificar quitação do contrato:
            Se todas NORMAL + INTERMEDIARIA pagas:
                contrato.status = QUITADO
                contrato.save()
    │
    ▼
Redirect /contratos/<id>/  ou  JSON {sucesso: true}
```
