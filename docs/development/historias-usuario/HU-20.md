# HU-20 — Notificações e Cobrança Automática

## História do Usuário

**Como** imobiliária  
**Quero** que o sistema envie automaticamente notificações de vencimento e inadimplência ao comprador por e-mail, SMS e WhatsApp, conforme uma régua de cobrança configurável  
**Para** reduzir inadimplência e eliminar o trabalho manual de cobrança individual

---

## Pré-condições

- Conta de e-mail SMTP configurada nos settings
- Provedor WhatsApp configurado (Evolution API, Twilio, Meta Cloud API ou Z-API) — opcional
- Comprador com e-mail e/ou telefone cadastrado

---

## Critérios de Aceitação

1. **N-01 — E-mail automático D-5**: 5 dias antes do vencimento, envio de e-mail de lembrete de cobrança via `enviar_notificacoes_sync()` em `core/tasks.py`
2. **N-02 — E-mail de inadimplência D+3**: 3 dias após vencimento sem pagamento, envio de alerta via `enviar_inadimplentes_sync()`; dias configurável por `NOTIFICACAO_DIAS_INADIMPLENCIA`
3. **N-03 — Régua de cobrança configurável**: modelo `RegraNotificacao` com `TipoGatilho` (ANTES / APÓS vencimento) e número de dias; permite criar regras customizadas por imobiliária
4. **N-04 — Integração multi-canal**: suporte a 4 provedores WhatsApp em `ConfiguracaoWhatsApp`:
   - Evolution API v2
   - Meta Cloud API (via Graph API)
   - Twilio WhatsApp
   - Z-API
5. **N-06 — Template unificado**: modelo `TemplateNotificacao` com 3 campos de conteúdo:
   - `corpo_html` → e-mail HTML (editor TinyMCE)
   - `corpo` → SMS (máximo 255 caracteres)
   - `corpo_whatsapp` → mensagem WhatsApp formatada
   - `renderizar()` → retorna 4-tuple: `(assunto, corpo, corpo_html, corpo_whatsapp)`
6. **N-07 — Validação de SMS**: formulário rejeita `corpo` > 255 caracteres; contador em tempo real com substituição das `%%TAGS%%` por valores de exemplo para exibir comprimento real
7. **N-08 — TEST_MODE safeguard**: em ambiente não-produção, todos os envios redirecionados para endereços de teste (`settings.EMAIL_TESTE`, `settings.TELEFONE_TESTE`) — nunca notifica compradores reais durante desenvolvimento
8. **N-09 — Normalização E.164**: telefones no formato `(31) 99999-8888` convertidos para `+5531999998888` antes do envio via Twilio
9. Disparo manual via `POST /financeiro/parcelas/<pk>/notificar/` → notifica comprador inadimplente individual
10. Jobs agendados via cron-job.org (não Celery — Render Free Tier não suporta workers):
    - N-01/N-02: `POST /api/tasks/enviar-notificacoes/` e `/api/tasks/enviar-inadimplentes/`
    - N-03: `_processar_regra()` avalia cada `RegraNotificacao` ativa

---

## Regras de Negócio

- RN-01: Deduplicação por `Notificacao` model — mesma parcela + mesmo tipo de gatilho não envia duas vezes no mesmo dia
- RN-02: Fallback automático: quando nenhuma `RegraNotificacao` configurada, usa N-01 (D-5) e N-02 (D+3) como padrão
- RN-03: `TipoGatilho.ANTES` → envia `dias` dias antes do vencimento; `TipoGatilho.APOS` → envia `dias` dias após
- RN-04: TEST_MODE ativado quando `settings.DEBUG=True` ou `settings.ENVIRONMENT != 'production'`
- RN-05: Normalização E.164 → strip de todos os não-dígitos + prefixo `+55` + número restante
- RN-06: Template único por `(codigo, imobiliaria)` — sem duplicatas por canal

## Definição de Pronto

- [ ] E-mail enviado D-5 antes do vencimento
- [ ] E-mail enviado D+3 após vencimento não pago
- [ ] Régua configurável por imobiliária funcional
- [ ] Templates com 3 canais (e-mail, SMS, WhatsApp)
- [ ] SMS bloqueado se > 255 caracteres
- [ ] TEST_MODE redireciona para endereços de teste
- [ ] Deduplicação impedindo envio duplicado no dia
- [ ] Jobs HTTP agendados no cron-job.org

## Cenários de Teste

### CT-01 — E-mail D-5 disparado
**Dado** parcela com vencimento em 5 dias, comprador com e-mail válido  
**Quando** job `/api/tasks/enviar-notificacoes/` roda  
**Então** e-mail enviado; `Notificacao` criada para deduplicação

### CT-02 — Deduplicação — não envia duas vezes
**Dado** e-mail D-5 já enviado hoje para a parcela  
**Quando** job roda novamente  
**Então** nenhum e-mail enviado; `Notificacao` existente impede reenvio

### CT-03 — TEST_MODE redireciona
**Dado** `DEBUG=True`, parcela com comprador email `comprador@real.com`  
**Quando** notificação disparada  
**Então** e-mail entregue em `settings.EMAIL_TESTE`, não para o comprador real

### CT-04 — SMS com > 255 caracteres bloqueado
**Dado** template com `corpo` de 280 caracteres  
**Quando** formulário enviado  
**Então** validação rejeita com mensagem de caracteres excedidos

### CT-05 — Régua customizada D+10
**Dado** `RegraNotificacao(tipo=APOS, dias=10, canal=WHATSAPP)` ativa  
**Quando** parcela vencida há 10 dias sem pagamento  
**Então** mensagem WhatsApp enviada ao comprador

### CT-06 — Normalização telefone
**Dado** comprador com `telefone = "(31) 99999-8888"`  
**Quando** notificação via Twilio é disparada  
**Então** envio para `+5531999998888`

## Fluxo do Processo

```
Cron-job.org (diário, 08:00 BRT)
    │
    ├──► POST /api/tasks/enviar-notificacoes/
    │           enviar_notificacoes_sync()
    │               ├── Busca parcelas com vencimento em D dias
    │               │   (D conforme RegraNotificacao.ANTES ativas)
    │               │   Fallback: D=5 (N-01)
    │               ├── Para cada parcela não paga:
    │               │       Se Notificacao(parcela, tipo, hoje) NÃO existe:
    │               │           TemplateNotificacao.renderizar(tags)
    │               │           Enviar e-mail / WhatsApp / SMS
    │               │           Notificacao.create(parcela, tipo, data=hoje)
    │               └── Retorna: {enviados, ignorados, erros}
    │
    └──► POST /api/tasks/enviar-inadimplentes/
                enviar_inadimplentes_sync()
                    ├── Busca parcelas vencidas há NOTIFICACAO_DIAS_INADIMPLENCIA (padrão 3)
                    │   (ou conforme RegraNotificacao.APOS ativas)
                    ├── Para cada parcela vencida não paga:
                    │       Se Notificacao(parcela, tipo_inadimplente, hoje) NÃO existe:
                    │           TemplateNotificacao.renderizar(tags + encargos calculados)
                    │           Enviar e-mail / WhatsApp / SMS
                    │           Notificacao.create(parcela, tipo_inadimplente, data=hoje)
                    └── Retorna: {enviados, ignorados, erros}

Operador (manual)
    │
    └──► POST /financeiro/parcelas/<pk>/notificar/
                Notifica individualmente 1 comprador inadimplente
```
