# HU-10 — Quitação via OFX (Extrato Bancário)

## História do Usuário

**Como** operador da imobiliária  
**Quero** importar o extrato bancário em formato OFX e conciliar automaticamente os pagamentos  
**Para** dar baixa em massa nas parcelas pagas via PIX, TED ou depósito sem digitação manual, com deduplicação automática

---

## Pré-condições

- Arquivo OFX (v1 ou v2) exportado do internet banking
- Parcelas em aberto no sistema
- Usuário com permissão de importação OFX

---

## Critérios de Aceitação

1. `POST /financeiro/cnab/ofx/upload/` recebe arquivo OFX e dispara processamento
2. `OFXService` faz parse via `POST /api/ofx/parse` na `boleto_cnab_api`; com **fallback automático para parser interno Python** em caso de indisponibilidade da API — processamento nunca é interrompido por falha da API externa
3. Cada transação do extrato é conciliada por **prioridade**:
   1. **Nosso número** no campo `MEMO` → match direto com `Parcela.nosso_numero_formatado`
   2. **Número do contrato** detectado no `MEMO`
   3. **Valor exato ± R$ 0,10** + janela de data (±5 dias do vencimento)
   4. **Valor genérico** → exibe sugestões para conciliação manual
4. Para cada match positivo, cria `HistoricoPagamento`:
   - `origem_pagamento = OFX`
   - `fitid_ofx = ID único da transação`
   - `forma_pagamento` derivada do tipo OFX
5. Segunda importação do mesmo OFX é idempotente — `fitid_ofx` duplicado é ignorado
6. Tela de resultado exibe: matches automáticos, conflitos (múltiplos candidatos), não conciliadas
7. Operador pode fazer conciliação manual para as transações não resolvidas automaticamente
8. Após importação: se todas as parcelas do contrato estiverem pagas, `status = QUITADO`
9. `GET /financeiro/cnab/retorno/` lista todos os arquivos importados com status e totais

---

## Regras de Negócio

- RN-01: `fitid_ofx` é único globalmente — mesma transação nunca gera pagamento duplicado
- RN-02: Transações de **débito** (saída) são ignoradas — apenas créditos viram pagamentos
- RN-03: Em caso de empate (duas parcelas com mesmo valor na mesma semana) → **não** concilia automaticamente; deixa para revisão manual
- RN-04: A baixa via OFX **não** é bloqueada por reajuste pendente
- RN-05: OFX corrompido ou formato não suportado → erro "Arquivo OFX inválido" sem processar nada

---

## Definição de Pronto

- [ ] `HistoricoPagamento` com `fitid_ofx` único persistido para cada match
- [ ] Segunda importação do mesmo OFX não duplica pagamentos
- [ ] Relatório final: matches automáticos, conflitos, pendências
- [ ] Transações de débito não geram pagamentos
- [ ] Contratos quitados automaticamente quando aplicável

---

## Cenários de Teste

### CT-01 — Match por nosso número no MEMO
**Dado** transação OFX com `MEMO = "PAG 000001234 REF CONTRATO"`  
**E** `Parcela.nosso_numero_formatado = "000001234"`  
**Quando** arquivo é processado  
**Então** match direto; `HistoricoPagamento` criado com `fitid_ofx` da transação

### CT-02 — Match por valor
**Dado** transação com `TRNAMT = 1500.00`, data 10/jan/2024  
**E** parcela com `valor_atual = 1500.00`, `data_vencimento = 05/jan/2024`  
**Quando** processado  
**Então** match por valor e janela de data; conciliado automaticamente

### CT-03 — Deduplicação — segundo OFX com mesma transação
**Dado** `fitid_ofx = "20240110-001"` já existente no banco  
**Quando** OFX reimportado com a mesma transação  
**Então** transação ignorada; nenhum `HistoricoPagamento` duplicado criado

### CT-04 — Transação de débito ignorada
**Dado** transação com `TRNTYPE = DEBIT`  
**Quando** processado  
**Então** transação ignorada sem criar pagamento

### CT-05 — Empate de match
**Dado** 2 parcelas com `valor_atual = 1.200,00` vencendo na mesma semana  
**E** transação OFX com `TRNAMT = 1200.00`  
**Quando** processado  
**Então** match marcado como "conflito"; operador deve resolver manualmente

### CT-06 — OFX corrompido
**Dado** arquivo com estrutura inválida  
**Quando** upload  
**Então** erro "Arquivo OFX inválido" e nenhum dado processado

## Fluxo do Processo

```
Operador exporta OFX do internet banking
    │
    ▼
POST /financeiro/cnab/ofx/upload/
    │   Arquivo OFX enviado
    │
    ├── OFXService.parse(arquivo)
    │       Tenta: POST /api/ofx/parse à boleto_cnab_api
    │       Fallback: parser interno se API indisponível
    │
    ▼
Para cada transação no OFX:
    │
    ├── Se DEBIT → ignorar
    │
    ├── Verificar fitid_ofx já existe → ignorar (deduplicação)
    │
    ├── Tentar match (em ordem):
    │   1. nosso_numero_formatado no MEMO
    │   2. numero_contrato no MEMO
    │   3. valor ± R$0,10 + janela de data
    │   4. → sem match: enfileirar para revisão manual
    │
    └── Se match único:
            HistoricoPagamento.create(
                origem_pagamento=OFX,
                fitid_ofx=fitid,
                valor_pago=trnamt
            )
            parcela.pago = True
            parcela.status_boleto = PAGO
            Verificar quitação do contrato
    │
    ▼
Exibir relatório:
    ├── Matches automáticos: N
    ├── Conflitos: N
    └── Não conciliadas: N
    │
    ▼ (opcional)
Conciliação manual das pendentes
    │
    ▼
Importação concluída ✅
```
