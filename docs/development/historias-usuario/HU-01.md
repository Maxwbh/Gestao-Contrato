# HU-01 — Criação de Contrato (Wizard)

## História do Usuário

**Como** operador da imobiliária  
**Quero** cadastrar um novo contrato de compra e venda em etapas guiadas  
**Para** garantir que todos os dados financeiros (amortização, juros, intermediárias) estejam configurados corretamente antes de gerar as parcelas

---

## Pré-condições

- Imobiliária cadastrada e ativa em `core.Imobiliaria`
- Imóvel disponível cadastrado em `imoveis.Imovel`
- Comprador cadastrado em `core.Comprador`
- Conta bancária ativa em `core.ContaBancaria` (necessária para emissão de boletos futura)
- Usuário autenticado com permissão de criação de contratos

---

## Critérios de Aceitação

1. O wizard está acessível em `GET /contratos/wizard/` e percorre as etapas `basico → juros → intermediarias → preview`
2. Dados são preservados na sessão `request.session['wizard_contrato']` entre etapas
3. **Etapa Básica** — campos obrigatórios:
   - `numero_contrato` (único por imobiliária)
   - `data_contrato`, `data_primeiro_vencimento`
   - `valor_total`, `valor_entrada` (entrada < total)
   - `numero_parcelas` (1–360), `dia_vencimento` (1–31)
   - `tipo_amortizacao`: `PRICE` (PMT constante) ou `SAC` (amortização constante)
   - `tipo_correcao`: IPCA / IGPM / INCC / IGPDI / INPC / TR / SELIC / FIXO
   - `prazo_reajuste_meses` (1–120, padrão 12)
4. **Etapa Juros** — tabela de juros escalante por ciclo (`TabelaJurosContrato`):
   - Cada linha: ciclo_início, ciclo_fim (null = válido indefinidamente), juros_mensal (%)
   - Etapa opcional; sem tabela o sistema opera em modo simples
5. **Etapa Intermediárias** — três modos:
   - `PADRAO`: intervalo regular (quantidade e meses entre elas)
   - `MANUAL`: lista de datas e valores individuais
   - `NENHUMA`: sem intermediárias
   - Validação: soma das intermediárias ≤ 80% do valor financiado
6. **Etapa Preview** — exibe antes do commit: PMT ciclo 1, PMT último ciclo, total intermediárias, valor financiado
7. `POST step=salvar` executa em transação atômica: `Contrato` + `TabelaJurosContrato` + `PrestacaoIntermediaria` + `gerar_parcelas()` + `recalcular_amortizacao()`
8. Após criação, redireciona para `/contratos/<id>/`

---

## Regras de Negócio

- RN-01: `numero_contrato` é único por imobiliária; duplicata rejeita o formulário
- RN-02: `valor_entrada` deve ser estritamente menor que `valor_total`
- RN-03: Soma das intermediárias ≤ 80% do valor financiado (`valor_total − valor_entrada`)
- RN-04: Quando `tipo_correcao = FIXO`, reajuste nunca é aplicado e boleto nunca é bloqueado
- RN-05: `bloqueio_boleto_reajuste` inicia como `False`; `ciclo_reajuste_atual` inicia como `1`
- RN-06: Amortização **PRICE** → `PMT = PV × i / (1 − (1+i)^−n)` onde `i = taxa_mensal/100`
- RN-07: Amortização **SAC** → amortização constante = `PV/n`; PMT decrescente
- RN-08: Ciclo de cada parcela: `ciclo = (numero_parcela − 1) ÷ prazo_reajuste_meses + 1`

---

## Definição de Pronto

- [ ] Contrato persistido com `status = ATIVO` e auditoria `created_at / updated_at`
- [ ] `numero_parcelas` parcelas geradas (HU-02 executada com sucesso)
- [ ] `TabelaJurosContrato` salva se informada
- [ ] `PrestacaoIntermediaria` criadas nos modos PADRAO ou MANUAL
- [ ] Amortização (`amortizacao`, `juros_embutido`) calculada por parcela se houver tabela de juros
- [ ] Wizard exibe preview antes do commit sem persistir dados
- [ ] Erro de validação exibido sem perder dados das etapas anteriores

---

## Cenários de Teste

### CT-01 — Contrato PRICE com Tabela de Juros, sem intermediárias
**Dado** que o operador preenche todos os campos da etapa básica com `tipo_amortizacao=PRICE`, `prazo=12`, `parcelas=120`  
**E** cadastra a tabela: ciclo 1 → 0%, ciclo 2 → 0,6%/mês, ciclo 7+ → 0,85%/mês  
**E** seleciona modo `NENHUMA` em intermediárias  
**Quando** confirma na etapa preview  
**Então** o contrato é criado com 120 parcelas, `amortizacao` e `juros_embutido` preenchidos

### CT-02 — Contrato SAC com intermediárias MANUAL
**Dado** `tipo_amortizacao=SAC` e duas intermediárias com datas e valores específicos  
**Quando** confirma  
**Então** parcelas têm `amortizacao` constante, `juros_embutido` decrescente e 2 `PrestacaoIntermediaria` criadas

### CT-03 — Valor de entrada inválido
**Dado** `valor_entrada = valor_total`  
**Quando** submete a etapa básica  
**Então** formulário rejeita com mensagem "Entrada não pode ser maior ou igual ao valor total"

### CT-04 — Número de contrato duplicado
**Dado** que já existe contrato com `numero_contrato = '001/2024'`  
**Quando** tenta criar outro com o mesmo número  
**Então** formulário rejeita com erro de unicidade

### CT-05 — Soma de intermediárias excede 80%
**Dado** valor financiado = R$ 100.000 e soma das intermediárias = R$ 85.000  
**Quando** avança para a etapa preview  
**Então** sistema rejeita com mensagem de validação

## Fluxo do Processo

```
Operador
    │
    ▼
GET /contratos/wizard/          → etapa: basico
    │   Preenche campos básicos
    ▼
POST /contratos/wizard/basico   → valida formulário
    │   Salva em sessão
    ▼
GET /contratos/wizard/juros     → etapa: tabela de juros
    │   (pode pular — nenhuma tabela)
    ▼
POST /contratos/wizard/juros    → salva tabela em sessão
    │
    ▼
GET /contratos/wizard/intermediarias → etapa: intermediárias
    │   Modo: PADRAO / MANUAL / NENHUMA
    ▼
POST /contratos/wizard/intermediarias → valida soma (≤80%)
    │
    ▼
GET /contratos/wizard/preview   → exibe resumo financeiro
    │   (PMT ciclo1, PMT final, total intermediárias)
    ▼
POST step=salvar                → TRANSAÇÃO ATÔMICA:
    │                               1. Criar Contrato
    │                               2. Criar TabelaJurosContrato
    │                               3. Criar PrestacaoIntermediaria
    │                               4. gerar_parcelas()
    │                               5. recalcular_amortizacao()
    ▼
Redirect /contratos/<id>/       → contrato criado ✅
```
