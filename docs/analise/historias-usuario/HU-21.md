# HU-21 — Portal do Comprador — Acesso e Autoatendimento Digital

## História do Usuário

**Como** comprador de um imóvel  
**Quero** acessar um portal web mobile-first com meu CPF/CNPJ e senha  
**Para** consultar meus contratos, baixar boletos, acompanhar meu histórico de pagamentos e atualizar meus dados sem precisar entrar em contato com a imobiliária

---

## Pré-condições

- Comprador cadastrado no sistema com CPF/CNPJ e e-mail
- Contrato ativo vinculado ao comprador
- Imobiliária sem restrição de acesso ao portal

---

## Critérios de Aceitação

### Auto-cadastro e Autenticação

1. `GET /portal/cadastro/` exibe formulário de auto-cadastro com CPF/CNPJ + e-mail + senha
2. Sistema valida que o CPF/CNPJ existe como `Comprador` no banco e que o e-mail confere com o cadastrado no contrato; rejeita caso contrário
3. Cria `User` Django com username `comprador_<documento>` e vincula via `AcessoComprador`
4. Se `AcessoComprador` já existe, redireciona para login
5. `POST /portal/login/` autentica via CPF/CNPJ + senha; se `AcessoComprador.ativo = False` bloqueia login mesmo com credenciais válidas
6. Cada login registra `LogAcessoComprador` (IP, user-agent, página, timestamp)

### Dashboard (`/portal/`)

7. Exibe KPIs consolidados de **todos os contratos** do comprador em uma única query:
   - Total de contratos ativos / parcelas pagas / pendentes / vencidas
   - Alerta de atraso (banner vermelho) se houver parcelas vencidas
   - Até 5 parcelas a vencer nos próximos 30 dias
   - Até 10 parcelas vencidas mais antigas
   - 5 últimas parcelas quitadas

### Meus Contratos (`/portal/contratos/`)

8. Lista todos os contratos do comprador (ATIVO, QUITADO, CANCELADO) com: número, imóvel, valor total, progresso (parcelas pagas / total) e status com badge colorido
9. Suporte a N contratos simultâneos por comprador — portal projetado para múltiplos contratos
10. `GET /portal/contratos/<id>/` exibe parcelas do contrato com: estatísticas, lista ordenada por número, intermediárias (se existirem), resumo financeiro via `get_resumo_financeiro()`, barra de progresso
11. Segurança: `get_object_or_404(..., comprador=comprador)` — comprador acessa apenas seus contratos

### Meus Boletos (`/portal/boletos/`)

12. Lista parcelas de todos os contratos com filtros: Todos / A pagar / Vencidos / Pagos; paginação de 20 itens
13. Botão de download visível para qualquer boleto com `nosso_numero` e status ≠ `NAO_GERADO` (GERADO, REGISTRADO, VENCIDO)
14. `GET /portal/boletos/<id>/download/` serve PDF como `attachment`; `GET /portal/boletos/<id>/visualizar/` serve como `inline`
15. Ambas as views verificam que a parcela pertence ao comprador autenticado

### Meus Dados (`/portal/meus-dados/`)

16. Permite atualizar: CEP, logradouro, número, complemento, bairro, cidade, estado, e-mail, telefone, celular e preferências de notificação (e-mail, SMS, WhatsApp)
17. Campo `nome` exibido como somente leitura — não pode ser alterado pelo comprador

### APIs JSON

18. `GET /portal/api/contratos/<id>/parcelas/` → lista de parcelas em JSON (id, número, vencimento, valor, pago, dias_atraso, tem_boleto)
19. `GET /portal/api/resumo-financeiro/` → resumo consolidado (total_pago, saldo_devedor, próximo_vencimento, parcelas_vencidas)
20. `GET /portal/api/vencimentos/` → próximos vencimentos ordenados com status; parcelas pagas excluídas
21. `GET /portal/api/segunda-via/<id>/` → linha digitável da parcela; parcela paga → 404; rate-limit 10 req/min

---

## Regras de Negócio

- RN-01: `AcessoComprador.ativo = False` bloqueia login e acesso ao portal — o admin desativa no painel Django
- RN-02: O e-mail informado no auto-cadastro deve coincidir com o `comprador.email` no sistema — impedimento de acesso não autorizado
- RN-03: Comprador com N contratos vê todos os contratos e parcelas no portal — não há restrição por contrato
- RN-04: CPF do comprador nunca é exibido em páginas públicas (RN-G05 global)
- RN-05: Todas as queries do dashboard usam `aggregate()` — sem N+1
- RN-06: `LogAcessoComprador` registra IP, user-agent, página e timestamp de cada login
- RN-07: Parcelas de intermediárias são exibidas separadamente na tela de detalhe do contrato

---

## Definição de Pronto

- [ ] Auto-cadastro por CPF/CNPJ + validação de e-mail do contrato
- [ ] Login/logout com `AcessoComprador.ativo` respeitado

- [ ] Lista de contratos com barra de progresso e status

- [ ] Atualização de dados pessoais e preferências de notificação

## Cenários de Teste

### CT-01 — Auto-cadastro bem-sucedido
**Dado** comprador com CPF `123.456.789-00` e e-mail `comprador@email.com` no sistema  
**Quando** preenche o formulário com CPF, mesmo e-mail e senha  
**Então** `AcessoComprador` criado; login automático; redirect para `/portal/`

### CT-02 — E-mail divergente no auto-cadastro
**Dado** comprador com e-mail `comprador@email.com` no contrato  
**Quando** tenta se cadastrar com `outro@email.com`  
**Então** formulário rejeitado com mensagem "E-mail não corresponde ao cadastro"

### CT-03 — Comprador bloqueado não acessa
**Dado** `AcessoComprador.ativo = False`  
**Quando** tenta login com CPF + senha corretos  
**Então** bloqueado com mensagem "Acesso bloqueado. Entre em contato com a imobiliária."

### CT-04 — Dashboard com N contratos
**Dado** comprador com 3 contratos (12 parcelas pagas, 5 vencidas, 8 pendentes)  
**Quando** acessa `/portal/`  
**Então** KPIs corretos agregados; banner de atraso exibido; lista das 5 parcelas mais urgentes

### CT-05 — Download de boleto apenas do próprio comprador
**Dado** comprador A autenticado, parcela pertencente ao comprador B  
**Quando** tenta `GET /portal/boletos/<id_de_B>/download/`  
**Então** 404 — comprador não acessa boleto de terceiro

### CT-06 — Linha digitável via API
**Dado** comprador autenticado, parcela com boleto gerado (`nosso_numero` preenchido)  
**Quando** `GET /portal/api/segunda-via/<id>/`  
**Então** JSON com linha digitável retornado

### CT-07 — Linha digitável indisponível para parcela paga
**Dado** parcela com `pago = True`  
**Quando** `GET /portal/api/segunda-via/<id>/`  
**Então** 404 — boleto de parcela paga não disponível

## Fluxo do Processo

```
Comprador → /portal/cadastro/
    │ CPF/CNPJ + e-mail + senha
    │
    ├── Busca Comprador por CPF/CNPJ → não encontrado → erro
    ├── E-mail confere? → não → erro
    ├── AcessoComprador já existe? → redirect /portal/login/
    └── Cria User + AcessoComprador → login automático
    │
    ▼
/portal/ (Dashboard)
    │   KPIs: aggregate(pagas, pendentes, vencidas, saldo)
    │   Próximas parcelas (30d) / Vencidas / Últimos pagamentos
    │
    ├──► /portal/contratos/
    │       Cards de cada contrato com progresso
    │       │
    │       └──► /portal/contratos/<id>/
    │               Parcelas do contrato + intermediárias
    │               get_resumo_financeiro()
    │
    ├──► /portal/boletos/
    │       Filtros: Todos / A pagar / Vencidos / Pagos
    │       Paginação 20/página
    │       │
    │       ├──► /portal/boletos/<id>/download/ (PDF attachment)
    │       └──► /portal/boletos/<id>/visualizar/ (PDF inline)
    │
    ├──► /portal/meus-dados/
    │       Atualiza endereço, e-mail, telefone, preferências
    │
    └──► APIs JSON
            /portal/api/contratos/<id>/parcelas/
            /portal/api/resumo-financeiro/
            /portal/api/vencimentos/
            /portal/api/segunda-via/<id>/ (rate-limit 10/min)
```
