# HU-02 — Geração de Parcelas

## História do Usuário

**Como** sistema  
**Quero** gerar automaticamente todas as parcelas do contrato ao ser criado  
**Para** ter o cronograma financeiro completo com vencimentos, valores e ciclos de reajuste corretos desde o início

---

## Pré-condições

- Contrato criado com `status = ATIVO` (HU-01 concluída)
- `numero_parcelas`, `data_primeiro_vencimento`, `dia_vencimento`, `tipo_amortizacao` definidos

---

## Critérios de Aceitação

1. Método `Contrato.gerar_parcelas()` cria exatamente `numero_parcelas` registros de `Parcela` com `tipo_parcela = NORMAL`
2. Além das parcelas normais, cria 1 parcela com `tipo_parcela = ENTRADA` se `valor_entrada > 0`
3. As `PrestacaoIntermediaria` são criadas como parcelas com `tipo_parcela = INTERMEDIARIA` nas datas configuradas
4. `data_vencimento` de cada parcela é calculada avançando meses a partir de `data_primeiro_vencimento` com ajuste inteligente para meses curtos (ex: 31/jan + 1 mês = 28/fev)
5. Ciclo de reajuste atribuído por parcela: `ciclo = (numero_parcela − 1) ÷ prazo_reajuste_meses + 1`
6. O endpoint `GET /contratos/wizard/api/preview-parcelas/` retorna a tabela simulada **sem persistir**
7. `Contrato.completar_parcelas_faltantes()` cria apenas as parcelas ausentes sem duplicar as existentes
8. Quando `gerar_parcelas(ate_mes_atual=True)` é usado (carga de dados), gera apenas até o mês atual; parcelas futuras são criadas posteriormente via `completar_parcelas_faltantes()`

---

## Regras de Negócio

- RN-01: Toda parcela inicia com `pago = False` e `status_boleto = NAO_GERADO`
- RN-02: `valor_original` e `valor_atual` iniciam iguais ao valor da parcela calculado
- RN-03: Para **PRICE**: `PMT = PV × i / (1 − (1+i)^−n)` — todas as parcelas têm mesmo `valor_original`
- RN-04: Para **SAC**: `amortizacao = PV / n` constante; `juros_embutido = saldo_atual × i`; PMT decrescente
- RN-05: Parcelas com `dia_vencimento = 31` em meses com menos de 31 dias vencem no último dia do mês
- RN-06: `completar_parcelas_faltantes()` verifica `numero_parcela` existente antes de criar — nunca duplica
- RN-07: O `token_publico` UUID é gerado automaticamente para cada parcela na criação

---

## Definição de Pronto

- [ ] `contrato.parcelas.count()` = `numero_parcelas` após criação
- [ ] Soma de `valor_original` das NORMAL + `valor_entrada` = `valor_total` (tolerância R$ 0,01)
- [ ] Todas as parcelas têm `ciclo` correto baseado em `prazo_reajuste_meses`
- [ ] Datas de vencimento respeitam ajuste para meses curtos
- [ ] `token_publico` único por parcela

---

## Cenários de Teste

### CT-01 — Geração completa PRICE com 120 parcelas
**Dado** contrato com `numero_parcelas=120`, `tipo_amortizacao=PRICE`, taxa 0,6%/mês  
**Quando** `gerar_parcelas()` é chamado  
**Então** 120 parcelas NORMAL com mesmo `valor_original` são criadas, `amortizacao` crescente e `juros_embutido` decrescente

### CT-02 — Vencimento dia 31 em fevereiro
**Dado** `data_primeiro_vencimento = 31/jan/2024`, `numero_parcelas = 3`  
**Quando** as parcelas são geradas  
**Então** datas são: 31/jan, 29/fev (ajustado), 31/mar

### CT-03 — Completar parcelas faltantes
**Dado** contrato com 120 parcelas onde apenas 24 foram geradas (modo `ate_mes_atual=True`)  
**Quando** `completar_parcelas_faltantes()` é chamado  
**Então** as 96 parcelas restantes são criadas sem duplicar as 24 existentes

### CT-04 — Ciclos de reajuste com prazo 12
**Dado** `prazo_reajuste_meses=12` e `numero_parcelas=36`  
**Quando** as parcelas são geradas  
**Então** parcelas 1–12 têm `ciclo=1`, 13–24 têm `ciclo=2`, 25–36 têm `ciclo=3`

### CT-05 — Preview sem persistir
**Dado** operador na etapa preview do wizard  
**Quando** `GET /contratos/wizard/api/preview-parcelas/` é chamado  
**Então** retorna JSON com a tabela calculada e nenhum registro é criado no banco

---

## Fluxo do Processo

```
Contrato salvo (HU-01)
    │
    ▼
Contrato.gerar_parcelas()
    │
    ├── Para cada número de 1 a numero_parcelas:
    │       1. Calcular data_vencimento = avançar mês + ajustar dia
    │       2. Calcular ciclo = (n−1) ÷ prazo + 1
    │       3. Calcular valor: PRICE (PMT constante) ou SAC (amortização constante)
    │       4. Criar Parcela(numero_parcela=n, tipo=NORMAL, pago=False, status=NAO_GERADO)
    │
    ├── Criar Parcela(tipo=ENTRADA, valor=valor_entrada) se entrada > 0
    │
    └── Para cada PrestacaoIntermediaria:
            Criar Parcela(tipo=INTERMEDIARIA, data=data_intermediaria, valor=valor_intermediaria)
    │
    ▼
recalcular_amortizacao() (se TabelaJurosContrato existe)
    │
    ├── Para cada parcela NORMAL não paga:
    │       Recalcular: amortizacao, juros_embutido, valor_original, valor_atual
    │
    ▼
Parcelas prontas ✅
    │
    ▼ (futuro, sob demanda)
completar_parcelas_faltantes()
    │
    ├── Busca max(numero_parcela) existente
    └── Cria parcelas de (max+1) até numero_parcelas
```
