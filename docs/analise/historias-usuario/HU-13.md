# HU-13 — Link Público de Boleto

## História do Usuário

**Como** comprador  
**Quero** acessar meu boleto através de um link seguro e opaco recebido por WhatsApp ou e-mail  
**Para** visualizar, copiar a linha digitável e fazer download do PDF sem precisar de login no sistema

---

## Pré-condições

- Parcela existente no sistema (token gerado automaticamente na criação)
- Boleto gerado (para download do PDF) — a página de visualização funciona mesmo sem PDF

---

## Critérios de Aceitação

1. Cada `Parcela` possui campo `token_publico` (UUID v4) gerado automaticamente em `default=uuid.uuid4`, imutável após criação
2. O link tem formato `/b/<uuid>/` — completamente opaco, sem expor IDs internos, nome do app, CPF ou estrutura do sistema
3. `GET /b/<uuid>/` → retorna HTTP 200 **sem exigir autenticação**
4. Token UUID inválido ou inexistente → HTTP 404
5. **Página pública exibe**:
   - Nome e logotipo da imobiliária
   - Nome do comprador
   - Número e referência da parcela
   - Data de vencimento
   - Valor (com juros calculados em tempo real se vencido)
   - Linha digitável (código de barras)
   - PIX copia e cola (se disponível)
   - Botão de download do PDF
6. **Página pública NÃO exibe**: CPF, CNPJ, agência/conta bancária, IDs de banco de dados, ações administrativas
7. `GET /b/<uuid>/download/` → serve PDF do boleto sem autenticação:
   - Prioridade: `boleto_pdf_db` (BinaryField, persiste em ambiente efêmero)
   - Fallback: `boleto_pdf` (FileField)
   - Sem PDF: HTTP 404
8. O administrador pode compartilhar o link pela tela interna de visualização do boleto (`/financeiro/parcelas/<id>/boleto/visualizar/`) via botão "Compartilhar":
   - Mobile/navegador com suporte: usa Web Share API (`navigator.share`)
   - Desktop/sem suporte: copia URL para clipboard com feedback visual ("Link copiado!")

---

## Regras de Negócio

- RN-01: `token_publico` é único por parcela (`unique=True`); parcelas diferentes sempre têm tokens diferentes
- RN-02: O token **não expira** — é permanente e vinculado à parcela
- RN-03: A URL `/b/<uuid>/` não requer cookie de sessão, token JWT, nem nenhuma credencial
- RN-04: Juros são calculados em tempo real na página pública se `data_vencimento < hoje`
- RN-05: CPF e dados sensíveis **nunca** aparecem no HTML, headers ou JSON da página pública
- RN-06: O link público é construído com `Parcela.get_link_publico()` → `reverse('boleto_publico:visualizar', kwargs={'token': self.token_publico})`

---

## Definição de Pronto

- [ ] `token_publico` gerado automaticamente para todas as parcelas existentes e novas
- [ ] `GET /b/<uuid>/` retorna 200 sem login
- [ ] CPF ausente do HTML da página pública (testado com conteúdo da response)
- [ ] `GET /b/<uuid>/download/` retorna PDF com `Content-Type: application/pdf`
- [ ] Token inválido retorna 404
- [ ] Sem PDF gerado: download retorna 404
- [ ] Botão "Compartilhar" funciona via Web Share API e clipboard

---

## Cenários de Teste

### CT-01 — Acesso público sem login
**Dado** parcela com `token_publico = <uuid>`  
**Quando** cliente acessa `GET /b/<uuid>/` sem estar autenticado  
**Então** HTTP 200 com página contendo nome do comprador, vencimento e valor

### CT-02 — CPF não exposto
**Dado** comprador com CPF "123.456.789-01"  
**Quando** cliente acessa `GET /b/<uuid>/`  
**Então** string "12345678901" **não** aparece no HTML da resposta

### CT-03 — Token inválido
**Dado** UUID gerado aleatoriamente que não pertence a nenhuma parcela  
**Quando** `GET /b/<uuid>/`  
**Então** HTTP 404

### CT-04 — Download PDF com boleto gerado
**Dado** parcela com `boleto_pdf_db = b'%PDF-1.4...'` preenchido  
**Quando** `GET /b/<uuid>/download/`  
**Então** HTTP 200, `Content-Type: application/pdf`, body contém bytes do PDF

### CT-05 — Download sem PDF
**Dado** parcela sem boleto gerado (`boleto_pdf_db = None`, `boleto_pdf = None`)  
**Quando** `GET /b/<uuid>/download/`  
**Então** HTTP 404

### CT-06 — Tokens únicos por parcela
**Dado** contrato com 10 parcelas  
**Quando** listamos `token_publico` de todas  
**Então** 10 tokens distintos (nenhum duplicado)

### CT-07 — Compartilhamento via clipboard
**Dado** administrador na tela interna `/financeiro/parcelas/<id>/boleto/visualizar/`  
**E** navegador sem suporte a `navigator.share`  
**Quando** clica em "Compartilhar"  
**Então** URL copiada para clipboard; feedback visual "Link copiado!" exibido

## Fluxo do Processo

```
Geração do link (administrador)
    │
    ├── Parcela criada → token_publico = uuid4() atribuído automaticamente
    │
    ├── GET /financeiro/parcelas/<id>/boleto/visualizar/
    │       link_publico = request.build_absolute_uri(parcela.get_link_publico())
    │       → "https://dominio.com/b/<uuid>/"
    │
    └── Botão "Compartilhar":
            if navigator.share → navigator.share({url: link_publico})
            else → navigator.clipboard.writeText(link_publico) + feedback visual
    │
    ▼ (comprador recebe link via WhatsApp/e-mail)

Acesso público (comprador)
    │
    ▼
GET /b/<uuid>/
    │
    ├── Buscar Parcela por token_publico
    │       → Se não encontrada: HTTP 404
    │
    ├── Se vencida: calcular juros em tempo real
    │       Parcela.calcular_juros_multa(date.today())
    │
    └── Renderizar boleto_publico.html (sem base.html):
            ├── Nome + logo da imobiliária
            ├── Nome do comprador (SEM CPF)
            ├── Número da parcela, vencimento, valor
            ├── Linha digitável + código de barras
            ├── PIX copia e cola
            └── Botão download PDF
    │
    ▼
GET /b/<uuid>/download/
    │
    ├── Buscar Parcela por token_publico
    ├── Se boleto_pdf_db → serve BinaryField como PDF
    ├── Elif boleto_pdf → serve FileField como PDF
    └── Else → HTTP 404
    │
    ▼
Content-Type: application/pdf ✅
```
