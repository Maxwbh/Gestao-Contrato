# HU-19 — Chatbot WhatsApp — Atendimento Automático ao Comprador

## História do Usuário

**Como** comprador  
**Quero** enviar uma mensagem WhatsApp para a imobiliária e receber automaticamente minha 2ª via de boleto, situação de atraso, confirmação de comprovante e resumo financeiro  
**Para** resolver minhas demandas de cobrança sem precisar ligar ou acessar um portal, a qualquer hora do dia

---

## Pré-condições

- Evolution API configurada e conectada à instância WhatsApp da imobiliária (`ConfiguracaoWhatsApp`)
- Webhook `POST /notificacoes/webhook/evolution/` recebendo mensagens
- Comprador cadastrado no sistema com número de telefone

---

## Critérios de Aceitação

### Fluxo A — Identificação do Cliente
1. Qualquer mensagem recebida com `fromMe=False` é roteada para `WhatsAppBotService.processar()`
2. Busca comprador pelo número E.164 do remetente
3. Se não encontrado → solicita CPF: *"Olá! Informe seu CPF para acessar seus boletos:"*
4. CPF recebido → busca comprador → salva sessão → exibe menu principal
5. Se encontrado → exibe menu principal diretamente

### Fluxo B — Menu Principal
6. Menu enviado com as opções:
   - `1️⃣ 2ª via de boleto`
   - `2️⃣ Boletos em atraso`
   - `3️⃣ Enviar comprovante de pagamento`
   - `4️⃣ Meu resumo financeiro`
   - `0️⃣ Falar com atendente`

### Fluxo C — 2ª Via de Boleto
7. Comprador digita "1", "segunda via", "2a via" ou "boleto"
8. Sistema lista até 5 parcelas PENDENTES + VENCIDAS com vencimento e valor
9. Comprador seleciona pelo número → `BoletoService.gerar_segunda_via()` chamado
10. Enviado: texto com linha digitável + PDF do boleto via `sendMedia` (Evolution API, base64)

### Fluxo D — Boletos em Atraso
11. Comprador digita "2", "atraso" ou "vencido"
12. Lista parcelas vencidas com encargos calculados pro rata die (principal + juros + multa + total hoje)
13. Permite selecionar parcela para receber 2ª via (continua no Fluxo C)

### Fluxo E — Receber Comprovante de Pagamento
14. Comprador digita "3", "comprovante", "paguei" ou envia imagem/PDF diretamente
15. Sistema solicita qual parcela o comprovante se refere; lista parcelas aguardando pagamento
16. Arquivo recebido (base64 via Evolution) → salvo em `HistoricoPagamento.comprovante` (FileField)
17. Cria `Notificacao` interna para admin revisar
18. Envia confirmação: *"✅ Comprovante recebido! Nossa equipe confirmará em até 1 dia útil."*
19. Notificação por e-mail enviada à imobiliária

### Fluxo F — Resumo Financeiro
20. Comprador digita "4", "saldo", "resumo" ou "situação"
21. Exibe: parcelas pagas/total, total pago, próximo vencimento, em atraso, progresso (%)
22. Dados via `Contrato.get_resumo_financeiro()`

### Opção Atendente
23. Digitar "0", "atendente", "humano" ou "falar com" → pausa o bot e notifica staff por e-mail

### Sessão e Timeout
24. Estado da sessão armazenado em `SessaoConversaWhatsApp` (banco de dados)
25. Sessões inativas há mais de 30 minutos são limpas pelo management command `limpar_sessoes_whatsapp`
26. Após 20 min sem resposta: aviso de timeout enviado ao comprador

---

## Regras de Negócio

- RN-01: Palavras-chave de cancelamento ("cancelar", "voltar", "sair", "menu") reiniciam para o menu principal
- RN-02: CPF inválido ou não encontrado → nova solicitação de CPF (máximo 3 tentativas)
- RN-03: `gerar_segunda_via()` gera PDF com encargos calculados para **hoje** — não usa PDF armazenado
- RN-04: Comprovante salvo em FileField; admin **deve** aprovar manualmente — bot não baixa automaticamente
- RN-05: Número E.164 normalizado antes da busca: `(31) 99999-8888` → `+5531999998888`
- RN-06: Bot ignora mensagens enviadas **pelo** sistema (`fromMe=True`)
- RN-07: Sessão contém: `telefone`, `comprador_fk`, `estado` (enum), `contexto_json`, `updated_at`

**Estados da Sessão (`SessaoConversaWhatsApp.Estado`):**

| Estado | Descrição |
|--------|-----------|
| `IDLE` | Aguardando início |
| `AGUARDANDO_CPF` | Solicitou CPF, aguardando resposta |
| `MENU_PRINCIPAL` | Menu exibido |
| `AGUARDANDO_PARCELA_2VIA` | Aguardando seleção de parcela para 2ª via |
| `AGUARDANDO_PARCELA_COMP` | Aguardando seleção de parcela para comprovante |
| `AGUARDANDO_COMPROVANTE` | Aguardando envio do arquivo comprovante |

**Quando** envia qualquer mensagem  
**Então** bot responde "Informe seu CPF..."

### CT-03 — 2ª Via enviada com PDF
**Dado** comprador identificado, digita "1", seleciona parcela  
**Quando** sistema processa  
**Então** linha digitável enviada como texto + PDF do boleto via `sendMedia`

### CT-04 — Boleto em atraso com encargos
**Dado** parcela vencida há 15 dias, taxa juros 1%/mês, multa 2%  
**Quando** comprador escolha opção "2"  
**Então** mensagem exibe: principal + juros calculados pro rata + multa + total hoje

### CT-05 — Comprovante recebido e admin notificado
**Dado** comprador no estado `AGUARDANDO_COMPROVANTE` envia imagem  
**Quando** bot processa  
**Então** arquivo salvo, `Notificacao` criada, e-mail enviado à imobiliária

### CT-06 — Sessão expirada
**Dado** sessão com `updated_at` > 30 min  
**Quando** management command `limpar_sessoes_whatsapp` roda  
**Então** sessão excluída; próxima mensagem reinicia do Fluxo A

## Fluxo do Processo

```
WhatsApp (comprador)
    │ Mensagem recebida
    ▼
POST /notificacoes/webhook/evolution/
    │ fromMe=False
    ▼
WhatsAppBotService.processar(telefone, mensagem)
    │
    ├── Buscar SessaoConversaWhatsApp por telefone
    │       Não existe → estado = IDLE
    │
    ├── IDLE: buscar Comprador por telefone E.164
    │       Não encontrado → estado = AGUARDANDO_CPF
    │                         enviar "Informe seu CPF"
    │       Encontrado → estado = MENU_PRINCIPAL
    │                    enviar menu
    │
    ├── AGUARDANDO_CPF: validar CPF recebido
    │       Válido → associar comprador → MENU_PRINCIPAL
    │       Inválido → nova solicitação (até 3 tentativas)
    │
    ├── MENU_PRINCIPAL: interpretar opção recebida
    │       "1" / keyword 2ª via → AGUARDANDO_PARCELA_2VIA
    │       "2" / keyword atraso → listar parcelas vencidas
    │       "3" / keyword comprovante → AGUARDANDO_PARCELA_COMP
    │       "4" / keyword resumo → enviar get_resumo_financeiro()
    │       "0" / keyword atendente → pausar bot, notificar staff
    │
    ├── AGUARDANDO_PARCELA_2VIA: receber número da parcela
    │       BoletoService.gerar_segunda_via(parcela, date.today())
    │       _enviar_pdf(pdf_bytes) → Evolution sendMedia
    │       Enviar linha digitável como texto
    │       → MENU_PRINCIPAL
    │
    ├── AGUARDANDO_PARCELA_COMP: receber número da parcela
    │       → AGUARDANDO_COMPROVANTE
    │
    └── AGUARDANDO_COMPROVANTE: receber mídia
            Baixar base64 → salvar FileField
            Criar Notificacao para admin
            Enviar e-mail imobiliária
            → MENU_PRINCIPAL
    │
    ▼
Sessão salva com novo estado
```
