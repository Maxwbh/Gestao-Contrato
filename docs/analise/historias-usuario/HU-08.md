# HU-08 — Segunda Via de Boleto

## História do Usuário

**Como** operador da imobiliária  
**Quero** gerar uma segunda via de boleto com encargos (juros e multa) calculados para o dia atual  
**Para** que o comprador em atraso pague o valor correto sem precisar de negociação manual dos encargos

---

## Pré-condições

- Parcela com `pago = False`
- Conta bancária configurada na imobiliária
- API `boleto_cnab_api` disponível

---

## Critérios de Aceitação

1. `GET /financeiro/parcelas/<id>/boleto/segunda-via/` → página de preview com: valor original, juros calculados para hoje, multa calculada para hoje, total a pagar
2. `GET /financeiro/parcelas/<id>/boleto/segunda-via/?download=1` → gera PDF fresco com encargos do dia e serve para download direto
3. A geração chama `BoletoService.gerar_segunda_via(parcela, conta_bancaria, data_referencia=hoje)`
4. O PDF gerado inclui os valores atualizados de juros e multa — não usa o PDF armazenado anteriormente
5. Parcela com `pago = True` → redireciona com mensagem "Parcela já paga — segunda via não disponível"
6. Não altera `status_boleto` da parcela ao gerar segunda via (é uma geração avulsa, não substituição)
7. O PDF gerado na segunda via **não** sobrescreve `boleto_pdf_db` da parcela original

---

## Regras de Negócio

- RN-01: Juros calculados por pro rata die: `juros = valor_atual × (taxa_mensal/100) × (dias_atraso/30)`
- RN-02: Multa aplicada uma única vez: `multa = valor_atual × percentual_multa/100`
- RN-03: Se não há atraso (`data_referencia ≤ data_vencimento`) → juros = 0, multa = 0
- RN-04: A `data_referencia` é sempre `date.today()` ao gerar — não aceita data futura
- RN-05: Segunda via não altera o `nosso_numero` existente — reutiliza o original para conciliação bancária

---

## Definição de Pronto

- [ ] Preview exibe breakdown: valor, juros, multa, total
- [ ] PDF com encargos atualizados gerado e servido com `Content-Type: application/pdf`
- [ ] Parcela paga redireciona com mensagem
- [ ] `boleto_pdf_db` original não é sobrescrito
- [ ] Cálculo de juros correto para qualquer número de dias de atraso

---

## Cenários de Teste

### CT-01 — Preview de segunda via em atraso
**Dado** parcela vencida há 30 dias, taxa juros 1%/mês, multa 2%  
**Quando** `GET .../segunda-via/`  
**Então** exibe: `juros = valor × 1%`, `multa = valor × 2%`, `total = valor + juros + multa`

### CT-02 — Download PDF com encargos
**Dado** cenário CT-01  
**Quando** `GET .../segunda-via/?download=1`  
**Então** PDF gerado com valores de hoje, `Content-Type: application/pdf`, status 200

### CT-03 — Parcela no prazo (sem encargos)
**Dado** parcela com vencimento amanhã  
**Quando** `GET .../segunda-via/`  
**Então** exibe: juros = R$ 0,00, multa = R$ 0,00, total = valor_original

### CT-04 — Parcela paga
**Dado** parcela com `pago = True`  
**Quando** `GET .../segunda-via/`  
**Então** redireciona com mensagem "Parcela já paga"

### CT-05 — `boleto_pdf_db` não sobrescrito
**Dado** parcela com `boleto_pdf_db` preenchido  
**Quando** gera segunda via  
**Então** `boleto_pdf_db` da parcela continua com o PDF original; segunda via é gerada ad-hoc

---

## Fluxo do Processo

```
Operador acessa "Segunda Via"
    │
    ▼
GET /financeiro/parcelas/<id>/boleto/segunda-via/
    │
    ├── Se parcela.pago == True → redirect com mensagem
    │
    ├── Calcular encargos para hoje:
    │       Parcela.calcular_juros_multa(data_referencia=hoje)
    │       → (juros_hoje, multa_hoje)
    │
    └── Renderiza preview:
            valor_original, juros_hoje, multa_hoje, total_hoje
    │
    ▼ (operador clica "Baixar PDF")
GET /financeiro/parcelas/<id>/boleto/segunda-via/?download=1
    │
    ├── BoletoService.gerar_segunda_via(
    │       parcela,
    │       conta_bancaria,
    │       data_referencia=date.today()
    │   )
    │       → Monta payload com valor_original + juros + multa
    │       → POST /api/<banco>/boleto à API
    │       → Recebe PDF com valores atualizados
    │
    ▼
HttpResponse(pdf_bytes, content_type='application/pdf')
    Content-Disposition: attachment; filename="segunda-via-<nosso_numero>.pdf"
    │
    ▼
Comprador recebe PDF atualizado ✅
```
