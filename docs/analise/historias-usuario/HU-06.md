# HU-06 — Bloqueio de Boleto por Reajuste

## História do Usuário

**Como** sistema  
**Quero** impedir automaticamente a emissão de boletos quando há reajuste pendente não aplicado  
**Para** evitar que o comprador receba cobranças com valores desatualizados e garantir conformidade contratual

---

## Pré-condições

- Contrato `ATIVO` com `tipo_correcao ≠ FIXO`
- Parcela no ciclo ≥ 2
- Ciclo de reajuste vencido e ainda não aplicado

---

## Critérios de Aceitação

1. Método `Parcela.pode_gerar_boleto()` retorna `(True, "")` ou `(False, motivo)` antes de qualquer geração
2. Regra de desbloqueio em cascata: para uma parcela no ciclo N, verifica todos os ciclos de 2 até N:
   - Se `hoje < data_vencimento_do_ciclo` → interrompe verificação, boleto liberado
   - Se existe `Reajuste(ciclo=k, aplicado=True)` → ciclo ok, avança
   - Se não existe reajuste aplicado → **bloqueia** com mensagem "Reajuste do ciclo k pendente"
3. `contrato.bloqueio_boleto_reajuste = True` pode bloquear globalmente todos os boletos do contrato
4. Tela `/financeiro/reajustes/pendentes/` lista todos os contratos com bloqueio ativo, agrupados por imobiliária
5. Ao aplicar reajuste (HU-05), o bloqueio do ciclo correspondente é automaticamente levantado
6. Boletos gerados com `force=True` ignoram o bloqueio (uso administrativo)
7. `GET /financeiro/parcelas/<id>/boleto/status/` retorna JSON indicando se bloqueado e o motivo

---

## Regras de Negócio

- RN-01: Ciclo 1 **nunca** bloqueia — parcelas do primeiro ciclo sempre podem ter boleto
- RN-02: `tipo_correcao = FIXO` **nunca** bloqueia — sem reajuste, sem bloqueio
- RN-03: Quitação manual (HU-09) e via OFX (HU-10) **não** são afetadas pelo bloqueio
- RN-04: Bloqueio é por ciclo — ao aplicar reajuste do ciclo 2, ciclo 3 só bloqueará quando seu período vencer
- RN-05: Verificação de "ciclo vencido": `data_vencimento_do_ciclo = data_contrato + (ciclo − 1) × prazo_reajuste_meses`

---

## Definição de Pronto

- [ ] `pode_gerar_boleto()` retorna motivo descritivo quando bloqueia
- [ ] Ciclo 1 e FIXO sempre retornam `True`
- [ ] Painel de pendentes exibe contratos corretamente agrupados
- [ ] Após aplicar reajuste, bloqueio é levantado para aquele ciclo
- [ ] `force=True` permite geração mesmo com bloqueio

---

## Cenários de Teste

### CT-01 — Ciclo 1 — sempre liberado
**Dado** parcela na posição 5 de contrato com prazo 12 (ciclo = 1)  
**Quando** `pode_gerar_boleto()` é chamado  
**Então** retorna `(True, "")`

### CT-02 — FIXO — nunca bloqueia
**Dado** contrato com `tipo_correcao = FIXO`, parcela no ciclo 3  
**Quando** `pode_gerar_boleto()` é chamado  
**Então** retorna `(True, "")`

### CT-03 — Ciclo 2 pendente bloqueia
**Dado** contrato com prazo 12, hoje é depois do aniversário do ciclo 2, reajuste do ciclo 2 não aplicado  
**E** parcela está no ciclo 2  
**Quando** tenta gerar boleto  
**Então** `pode_gerar_boleto()` retorna `(False, "Reajuste do ciclo 2 pendente")`

### CT-04 — Verificação cascata: ciclo 2 ok, ciclo 3 pendente
**Dado** ciclo 2 aplicado, ciclo 3 não aplicado e vencido  
**E** parcela está no ciclo 3  
**Quando** `pode_gerar_boleto()` é chamado  
**Então** retorna `(False, "Reajuste do ciclo 3 pendente")`

### CT-05 — Aplicar reajuste levanta bloqueio
**Dado** boleto bloqueado por ciclo 2 pendente  
**Quando** operador aplica reajuste ciclo 2 (HU-05)  
**Então** `pode_gerar_boleto()` passa a retornar `(True, "")` para parcelas do ciclo 2

---

## Fluxo do Processo

```
Tentativa de gerar boleto (HU-03)
    │
    ▼
Parcela.pode_gerar_boleto()
    │
    ├── Se contrato.tipo_correcao == FIXO → return (True, "")
    │
    ├── Se parcela.ciclo == 1 → return (True, "")
    │
    └── Para k de 2 até parcela.ciclo:
            data_vencimento_ciclo = data_contrato + (k−1) × prazo_meses
            │
            ├── Se hoje < data_vencimento_ciclo → break (futuro, ok)
            │
            ├── Se Reajuste.objects.filter(ciclo=k, aplicado=True).exists()
            │       → continue (reajuste ok)
            │
            └── Else → return (False, f"Reajuste do ciclo {k} pendente")
    │
    ▼
(True, "") → prossegue para BoletoService
(False, motivo) → retorna JSON {sucesso: false, erro: motivo}

    │
    ▼ (resolução pelo operador)
/financeiro/reajustes/pendentes/
    │   Lista contratos bloqueados por imobiliária
    │
    ▼
Operador aplica reajuste (HU-05)
    │
    ▼
Bloqueio levantado ✅
```
