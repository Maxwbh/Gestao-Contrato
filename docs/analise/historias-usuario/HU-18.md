# HU-18 — Relatórios Financeiros

## História do Usuário

**Como** gestor da contabilidade ou operador da imobiliária  
**Quero** visualizar e exportar relatórios financeiros consolidados — prestações a pagar, pagas, posição dos contratos e previsão de reajustes  
**Para** monitorar a saúde financeira dos contratos e atender obrigações contábeis

---

## Pré-condições

- Usuário autenticado com permissão de visualizar relatórios
- Dados de contratos e parcelas disponíveis no sistema

---

## Critérios de Aceitação

1. `GET /financeiro/relatorios/prestacoes-a-pagar/` → relatório de parcelas em aberto com filtros por imobiliária, contrato, período de vencimento
2. `GET /financeiro/relatorios/prestacoes-pagas/` → relatório de parcelas já pagas com filtros por período de pagamento
3. `GET /financeiro/relatorios/posicao-contratos/` → snapshot da posição financeira de todos os contratos ativos: valor total, pago, a pagar, saldo devedor, progresso
4. `GET /financeiro/relatorios/previsao-reajustes/` → contratos com reajuste previsto nos próximos N meses, com índice e percentual estimado
5. `GET /financeiro/relatorios/exportar/<tipo>/` → exporta o relatório ativo em formato CSV, PDF ou Excel
6. `POST /financeiro/relatorios/exportar-consolidado/` → exporta relatório consolidado de múltiplas imobiliárias
7. `GET /financeiro/api/relatorios/resumo/` → JSON com totais resumidos para o dashboard
8. **Dashboard por Imobiliária**: `GET /financeiro/imobiliaria/<id>/dashboard/` → gráfico de Fluxo de Caixa Previsto vs Realizado com 12 meses (-5 passados a +6 futuros), renderizado com Chart.js
9. `GET /financeiro/api/imobiliaria/<id>/fluxo-caixa/` → JSON com 3 séries de dados: `Previsto`, `Realizado`, `Pendente`

---

## Regras de Negócio

- RN-01: Relatório "A Pagar" inclui apenas parcelas com `pago = False` e `data_vencimento ≤ data_limite_filtro`
- RN-02: Relatório "Pagas" inclui `HistoricoPagamento` agrupado por data de pagamento
- RN-03: Posição dos contratos considera `saldo_devedor` pelo método correto (PRICE ou SAC)
- RN-04: Fluxo de caixa `Previsto` = parcelas não pagas com vencimento no período; `Realizado` = pagamentos efetuados; `Pendente` = vencidas não pagas
- RN-05: Export Excel usa `openpyxl`; PDF usa WeasyPrint ou ReportLab conforme configuração

---

## Definição de Pronto

- [ ] 4 relatórios tabulares disponíveis com filtros
- [ ] Exportação em CSV e PDF funcional
- [ ] Dashboard de fluxo de caixa com 12 meses exibido por imobiliária
- [ ] JSON de fluxo de caixa com 3 séries (Previsto, Realizado, Pendente)
- [ ] Permissões corretas por perfil (gestor vs operador)

---

## Cenários de Teste

### CT-01 — Relatório a pagar filtrado por mês
**Dado** 50 parcelas vencendo em maio/2026  
**Quando** `GET /financeiro/relatorios/prestacoes-a-pagar/?mes=5&ano=2026`  
**Então** retorna 50 registros com valor total correto

### CT-02 — Exportar CSV de parcelas a pagar
**Dado** relatório a pagar com filtros aplicados  
**Quando** `GET /financeiro/relatorios/exportar/csv/`  
**Então** arquivo `.csv` com cabeçalhos e dados corretos, `Content-Disposition: attachment`

### CT-03 — Dashboard fluxo de caixa
**Dado** imobiliária com contratos ativos em 2026  
**Quando** `GET /financeiro/api/imobiliaria/<id>/fluxo-caixa/`  
**Então** JSON com 12 pontos por série, valores de Previsto/Realizado/Pendente para cada mês

### CT-04 — Previsão de reajustes
**Dado** 5 contratos com aniversário de reajuste nos próximos 3 meses  
**Quando** `GET /financeiro/relatorios/previsao-reajustes/?meses=3`  
**Então** lista os 5 contratos com data prevista, índice e percentual estimado

## Fluxo do Processo

```
Operador acessa Relatórios
    │
    ├──► Prestações a Pagar:
    │       GET /financeiro/relatorios/prestacoes-a-pagar/
    │           Filtros: imobiliária, contrato, período vencimento, status
    │           Exibe tabela paginada
    │           → GET /financeiro/relatorios/exportar/csv/ (ou pdf/excel)
    │
    ├──► Prestações Pagas:
    │       GET /financeiro/relatorios/prestacoes-pagas/
    │           Filtros: período de pagamento, forma de pagamento
    │
    ├──► Posição dos Contratos:
    │       GET /financeiro/relatorios/posicao-contratos/
    │           Por contrato: valor_total, pago, a_pagar, saldo_devedor, progresso%
    │
    ├──► Previsão de Reajustes:
    │       GET /financeiro/relatorios/previsao-reajustes/
    │           Contratos com aniversário no período + índice estimado
    │
    └──► Dashboard por Imobiliária:
            GET /financeiro/imobiliaria/<id>/dashboard/
                │
                ├── GET /financeiro/api/imobiliaria/<id>/fluxo-caixa/
                │       → JSON: {
                │           meses: [jan, fev, ..., dez],
                │           previsto: [R$, R$, ...],
                │           realizado: [R$, R$, ...],
                │           pendente: [R$, R$, ...]
                │         }
                │
                └── Chart.js renderiza gráfico de barras/linha com 3 séries
```
